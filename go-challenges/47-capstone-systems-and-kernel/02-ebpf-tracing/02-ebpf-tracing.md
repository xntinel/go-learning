# 2. eBPF Tracing Tool

eBPF (extended Berkeley Packet Filter) is a Linux kernel technology that runs verified bytecode in kernel context without modifying kernel source or loading kernel modules. A Go eBPF tracing tool must bridge two worlds: a restricted C program compiled for the BPF virtual machine that executes in kernel space, and a Go user-space program that loads that bytecode, attaches it to probe points, reads events from shared maps, and presents telemetry. The hard parts are: the kernel verifier rejects programs it cannot statically prove safe; kernel-to-userspace communication uses lock-free ring buffers where both sides must agree on binary struct layout byte-for-byte; and the tool needs `CAP_BPF`/`CAP_PERFMON` privileges (Linux 5.8+) or root.

```text
tracer/
  go.mod
  events.go          (event types + decode functions; stdlib only)
  histogram.go       (log2 latency histogram; stdlib only)
  tracer.go          (//go:generate directive + Tracer type; requires cilium/ebpf)
  tracer_test.go     (table-driven tests; stdlib only for event/histogram tests)
  cmd/demo/main.go   (CLI: openat | exec | histogram)
  bpf/tracer.bpf.c  (eBPF C program compiled to tracer.bpf.o by clang via bpf2go)
```

`events.go` and `histogram.go` are pure stdlib and testable on any platform. `tracer.go` requires Linux 5.8+ and `CAP_BPF`/`CAP_PERFMON` (or root) to load BPF objects and attach probes.

## Concepts

### The eBPF Execution Model

An eBPF program is bytecode compiled from restricted C by clang targeting the BPF architecture (`-target bpf`). Before any instruction runs, the kernel verifier performs exhaustive static analysis: every code path must terminate, every memory access must be bounds-checked, and every function call must use an approved kernel helper (`bpf_probe_read_kernel`, `bpf_ringbuf_reserve`, `bpf_ktime_get_ns`, etc.). Programs that pass are JIT-compiled to native machine code. The full lifecycle:

1. Compile: `clang -O2 -g -target bpf -c bpf/tracer.bpf.c -o tracer.bpf.o`
2. Parse the ELF in user space — `bpf2go` does this and generates Go bindings.
3. Create maps (ring buffers, hash tables, arrays) in the kernel via the `bpf()` syscall.
4. Load programs into the kernel; the verifier runs at this point and returns an error for unsafe code.
5. Attach programs to probe points: kprobe, kretprobe, tracepoint, XDP, uprobe.
6. Read events from ring buffers in a loop; decode with `encoding/binary`.
7. On exit, close all file descriptors; the kernel removes the programs automatically.

### Maps: Kernel-to-Userspace Channels

eBPF maps are key/value stores allocated in kernel memory and accessible from both sides via file descriptors. The important types for a tracing tool:

- `BPF_MAP_TYPE_HASH` — arbitrary key/value, O(1) average. Used to store per-PID entry timestamps for latency measurement.
- `BPF_MAP_TYPE_ARRAY` — integer-indexed, pre-allocated, fixed size. Used for log2 histogram buckets.
- `BPF_MAP_TYPE_RINGBUF` (kernel 5.8+) — lock-free single-producer/single-consumer ring buffer. The kernel side reserves space, writes the record, and commits atomically. The consumer (Go) polls or blocks. Preferred over `PERF_EVENT_ARRAY` for new code because it avoids per-CPU buffer fragmentation.

Both sides must agree on the binary layout of each record. Always use fixed-width integer types (`__u32`, `__u64` in C; `uint32`, `uint64` in Go) and account for alignment padding: a `__u32` followed immediately by a `__u64` in C will have 4 bytes of padding inserted by the compiler; the Go struct must include a matching blank or named padding field. The cleanest approach is to design C structs so alignment padding is not needed — for example, two `__u32` fields before a `__u64` naturally fill 8 bytes.

### The bpf2go Workflow

`bpf2go` (part of the cilium/ebpf module) compiles C and generates Go bindings in one step, invoked through `go generate`:

```go
//go:generate go run github.com/cilium/ebpf/cmd/bpf2go -cc clang tracer bpf/tracer.bpf.c
```

Running `go generate ./...` produces `tracer_bpfel.go` (little-endian) and `tracer_bpfeb.go` (big-endian). Each contains:

- The compiled `.o` file embedded as a byte slice — no external file needed at runtime.
- A `tracerObjects` struct with one typed field per map and program.
- `loadTracerObjects(objs *tracerObjects, opts *ebpf.CollectionOptions) error`.

The embed means the shipping binary is self-contained. The generated type for the programs and maps used in this lesson looks like:

```go
// Generated by bpf2go — do not edit.

	tracerMaps
```

### Probe Types

| Probe | Kernel mechanism | Use in this lesson |
|---|---|---|
| kprobe | software breakpoint on any non-inlined kernel function | `sys_openat` entry, `vfs_read` entry |
| kretprobe | fires on function return; captures elapsed time | `vfs_read` return for latency |
| tracepoint | stable versioned hook points in kernel source | `sched:sched_process_exec` |
| uprobe | software breakpoint in a user-space binary | not covered here; see libbpf uprobe docs |
| XDP | runs in the network driver before kernel SKB allocation | not covered here; see cilium/ebpf XDP example |

Tracepoints are preferred over kprobes in production: their format is defined in `<tracefs>/events/.../format` and stable across kernel versions. kprobes break when the target function is renamed or inlined by the compiler.

### Log2 Latency Histograms

A log2 histogram covers the full nanosecond-to-second range with 64 buckets: bucket `i` holds samples with latency in `[2^i, 2^(i+1)-1]` nanoseconds (bucket 0 covers 0-1 ns). A 1 µs read (1000 ns) lands in bucket 9 because 2^9 = 512 and 2^10 = 1024. The BPF kernel side increments a 64-element array map; the Go side reads it and renders an ASCII bar chart. The Go `math/bits.LeadingZeros64` function computes the bucket index in O(1) using the hardware `CLZ` instruction.

### Privilege Requirements

The `bpf()` syscall requires one of: root (`uid == 0`), or `CAP_BPF` + `CAP_PERFMON` (Linux 5.8+, sufficient for tracing). XDP attachment additionally requires `CAP_NET_ADMIN`. Running without these returns `EPERM` from the `bpf()` syscall. Older kernels (before 5.11) also require raising the `RLIMIT_MEMLOCK` limit; `rlimit.RemoveMemlock()` from the cilium/ebpf package handles this automatically and is a no-op on 5.11+.

```go
package tracer

import (
	"bytes"
	"encoding/binary"
	"errors"
	"fmt"
	"time"

// Sentinel errors.
var (
	// ErrNotRoot is returned by New when the process lacks CAP_BPF+CAP_PERFMON or root.
	ErrNotRoot = errors.New("tracer: insufficient privileges; run as root or with CAP_BPF+CAP_PERFMON")

	// ErrKernelTooOld is returned by New when the kernel is older than 5.8.
	ErrKernelTooOld = errors.New("tracer: kernel 5.8+ required for BPF_MAP_TYPE_RINGBUF")

	// ErrClosed is returned by ReadOpenat and ReadExec after Close has been called.
	ErrClosed = errors.New("tracer: reader is closed")
)

// OpenatEvent mirrors the C struct openat_event in bpf/tracer.bpf.c.
// Field layout (little-endian, no implicit padding):
//
//	struct openat_event {
//	    __u32 pid;
//	    __u32 uid;
//	    __u64 ts_ns;
//	    char  filename[256];
//	};
//
// Two __u32 fields before __u64 means no padding is inserted by the C compiler.
type OpenatEvent struct {
	PID      uint32
	UID      uint32
	TsNs     uint64
	Filename [256]byte
}

// FilenameStr returns the null-terminated filename as a Go string.
func (e *OpenatEvent) FilenameStr() string {
	n := 0
	for n < len(e.Filename) && e.Filename[n] != 0 {
		n++
	}
	return string(e.Filename[:n])
}

// ExecEvent mirrors the C struct exec_event in bpf/tracer.bpf.c.
// Field layout (little-endian, no implicit padding):
//
//	struct exec_event {
//	    __u32 pid;
//	    __u32 ppid;
//	    __u64 ts_ns;
//	    char  comm[16];
//	    char  filename[256];
//	};
type ExecEvent struct {
	PID      uint32
	PPID     uint32
	TsNs     uint64
	Comm     [16]byte
	Filename [256]byte
}

// CommStr returns the null-terminated process name as a Go string.
func (e *ExecEvent) CommStr() string {
	n := 0
	for n < len(e.Comm) && e.Comm[n] != 0 {
		n++
	}
	return string(e.Comm[:n])
}

// FilenameStr returns the null-terminated filename as a Go string.
func (e *ExecEvent) FilenameStr() string {
	n := 0
	for n < len(e.Filename) && e.Filename[n] != 0 {
		n++
	}
	return string(e.Filename[:n])
}

// DecodeOpenat deserializes a raw ring buffer record into an OpenatEvent.
// raw must contain at least binary.Size(OpenatEvent{}) bytes (272 bytes).
func DecodeOpenat(raw []byte) (OpenatEvent, error) {
	var ev OpenatEvent
	want := binary.Size(ev)
	if len(raw) < want {
		return ev, fmt.Errorf("tracer: openat record too short: got %d bytes, want %d", len(raw), want)
	}
	if err := binary.Read(bytes.NewReader(raw[:want]), binary.LittleEndian, &ev); err != nil {
		return ev, fmt.Errorf("tracer: decode openat: %w", err)
	}
	return ev, nil
}

// DecodeExec deserializes a raw ring buffer record into an ExecEvent.
// raw must contain at least binary.Size(ExecEvent{}) bytes (288 bytes).
func DecodeExec(raw []byte) (ExecEvent, error) {
	var ev ExecEvent
	want := binary.Size(ev)
	if len(raw) < want {
		return ev, fmt.Errorf("tracer: exec record too short: got %d bytes, want %d", len(raw), want)
	}
	if err := binary.Read(bytes.NewReader(raw[:want]), binary.LittleEndian, &ev); err != nil {
		return ev, fmt.Errorf("tracer: decode exec: %w", err)
	}
	return ev, nil
}

// Timestamp converts a kernel CLOCK_MONOTONIC nanosecond value to a duration
// relative to startNs. Returns 0 when eventNs <= startNs (can occur near tracing start).
func Timestamp(startNs, eventNs uint64) time.Duration {
	if eventNs <= startNs {
		return 0
	}
	return time.Duration(eventNs-startNs) * time.Nanosecond
}
```

The struct comment explains why no padding field is needed: two `__u32` fields (`pid`, `uid`) fill 8 bytes, naturally aligning `ts_ns` at offset 8 — the same rule the C compiler applies.

```c
// bpf/tracer.bpf.c
//
// Compiled by bpf2go (invoked via go generate). To compile manually:
//   clang -O2 -g -target bpf -D__TARGET_ARCH_x86_64 \
//         -I$(bpftool btf dump file /sys/kernel/btf/vmlinux format c > vmlinux.h) \
//         -c bpf/tracer.bpf.c -o tracer.bpf.o
//
//go:build ignore

#include "vmlinux.h"
#include <bpf/bpf_helpers.h>
#include <bpf/bpf_tracing.h>
#include <bpf/bpf_core_read.h>

char LICENSE[] SEC("license") = "GPL";

/* openat_event must match tracer.OpenatEvent in events.go byte-for-byte. */
struct openat_event {
	__u32 pid;
	__u32 uid;
	__u64 ts_ns;
	char  filename[256];
};

struct {
	__uint(type, BPF_MAP_TYPE_RINGBUF);
	__uint(max_entries, 256 * 1024);
} openat_events SEC(".maps");

SEC("kprobe/sys_openat")
int kprobe_openat(struct pt_regs *ctx)
{
	struct openat_event *e = bpf_ringbuf_reserve(&openat_events, sizeof(*e), 0);
	if (!e)
		return 0;

	e->pid   = bpf_get_current_pid_tgid() >> 32;
	e->uid   = bpf_get_current_uid_gid() & 0xFFFFFFFF;
	e->ts_ns = bpf_ktime_get_ns();
	/* PT_REGS_PARM2 is the second argument: the userspace filename pointer. */
	bpf_probe_read_user_str(e->filename, sizeof(e->filename),
				(const void *)PT_REGS_PARM2(ctx));
	bpf_ringbuf_submit(e, 0);
	return 0;
}

/* exec_event must match tracer.ExecEvent in events.go byte-for-byte. */
struct exec_event {
	__u32 pid;
	__u32 ppid;
	__u64 ts_ns;
	char  comm[16];
	char  filename[256];
};

struct {
	__uint(type, BPF_MAP_TYPE_RINGBUF);
	__uint(max_entries, 256 * 1024);
} exec_events SEC(".maps");

SEC("tracepoint/sched/sched_process_exec")
int tracepoint_sched_process_exec(struct trace_event_raw_sched_process_exec *ctx)
{
	struct exec_event *e = bpf_ringbuf_reserve(&exec_events, sizeof(*e), 0);
	if (!e)
		return 0;

	struct task_struct *task = (struct task_struct *)bpf_get_current_task();
	e->pid   = bpf_get_current_pid_tgid() >> 32;
	e->ppid  = BPF_CORE_READ(task, real_parent, tgid);
	e->ts_ns = bpf_ktime_get_ns();
	bpf_get_current_comm(e->comm, sizeof(e->comm));
	bpf_probe_read_kernel_str(e->filename, sizeof(e->filename), ctx->filename);
	bpf_ringbuf_submit(e, 0);
	return 0;
}

/* --- vfs_read latency histogram --- */

struct {
	__uint(type, BPF_MAP_TYPE_HASH);
	__uint(max_entries, 65536);
	__type(key, __u32);
	__type(value, __u64);
} entry_times SEC(".maps");

struct {
	__uint(type, BPF_MAP_TYPE_ARRAY);
	__uint(max_entries, 64);
	__type(key, __u32);
	__type(value, __u64);
} latency_hist SEC(".maps");

SEC("kprobe/vfs_read")
int kprobe_vfs_read(struct pt_regs *ctx)
{
	__u32 pid = bpf_get_current_pid_tgid() >> 32;
	__u64 ts  = bpf_ktime_get_ns();
	bpf_map_update_elem(&entry_times, &pid, &ts, BPF_ANY);
	return 0;
}

SEC("kretprobe/vfs_read")
int kretprobe_vfs_read(struct pt_regs *ctx)
{
	__u32 pid   = bpf_get_current_pid_tgid() >> 32;
	__u64 *start = bpf_map_lookup_elem(&entry_times, &pid);
	if (!start)
		return 0;

	__u64 delta = bpf_ktime_get_ns() - *start;
	bpf_map_delete_elem(&entry_times, &pid);

	/* log2(delta): shift right until <= 1, counting shifts. */
	__u32 bucket = 0;
	__u64 v = delta;
	while (v > 1 && bucket < 63) {
		v >>= 1;
		bucket++;
	}
	__u64 *count = bpf_map_lookup_elem(&latency_hist, &bucket);
	if (count)
		__sync_fetch_and_add(count, 1);
	return 0;
}
```

### Exercise 2: Latency Histogram

Create `histogram.go`. This is pure stdlib and fully testable without a Linux kernel:

```go
package tracer

import (
	"fmt"
	"math/bits"
	"strings"
)

// histBuckets is the number of log2 buckets in a Histogram.
// Bucket i covers [2^i, 2^(i+1)-1] nanoseconds; 64 buckets span 1 ns to ~580 years.
const histBuckets = 64

// Histogram is a log2 latency histogram. It is not safe for concurrent use;
// the caller must synchronize if ReadLatencyHistogram and Record run concurrently.
type Histogram struct {
	counts [histBuckets]uint64
	total  uint64
}

// Record increments the bucket corresponding to a latency value in nanoseconds.
func (h *Histogram) Record(ns uint64) {
	h.counts[log2Bucket(ns)]++
	h.total++
}

// Counts returns a copy of the 64-element bucket count array.
// counts[i] is the number of samples with latency in [2^i, 2^(i+1)-1] ns.
func (h *Histogram) Counts() [histBuckets]uint64 {
	return h.counts
}

// Total returns the total number of recorded samples.
func (h *Histogram) Total() uint64 {
	return h.total
}

// String renders the histogram as an ASCII bar chart.
// Only non-empty buckets are printed. The bar width is proportional to the
// maximum count across all buckets.
func (h *Histogram) String() string {
	if h.total == 0 {
		return "(empty histogram)"
	}
	var maxCount uint64
	for _, c := range h.counts {
		if c > maxCount {
			maxCount = c
		}
	}
	const barWidth = 40
	var sb strings.Builder
	fmt.Fprintf(&sb, "%-22s %10s  distribution\n", "latency (ns)", "count")
	for i, c := range h.counts {
		if c == 0 {
			continue
		}
		lo := uint64(1) << uint(i)
		var hi uint64
		if i < 63 {
			hi = (uint64(1) << uint(i+1)) - 1
		} else {
			hi = ^uint64(0)
		}
		if i == 0 {
			lo = 0
			hi = 1
		}
		bar := int(float64(c) / float64(maxCount) * barWidth)
		fmt.Fprintf(&sb, "[%9d,%9d] %10d  |%s|\n",
			lo, hi, c, strings.Repeat("*", bar))
	}
	return sb.String()
}

// log2Bucket returns the zero-indexed log2 bucket for v.
// v=0 → 0; v=1 → 0; v=2 → 1; v=3 → 1; v=4 → 2; etc.
// Uses the hardware CLZ instruction via bits.LeadingZeros64.
func log2Bucket(v uint64) int {
	if v == 0 {
		return 0
	}
	b := 63 - bits.LeadingZeros64(v)
	if b >= histBuckets {
		return histBuckets - 1
	}
	return b
}
```

Verify the bucket assignment for common latencies: 500 ns is 2^8.97, so `63 - LeadingZeros64(500) = 63 - 55 = 8`. That matches the range [256, 511] ns. 1000 ns lands in bucket 9 ([512, 1023] ns).

### Exercise 3: Tracer Type

Create `tracer.go`. This file requires cilium/ebpf and the bpf2go-generated `tracerObjects` type:

```go
//go:generate go run github.com/cilium/ebpf/cmd/bpf2go -cc clang tracer bpf/tracer.bpf.c

package tracer

import (
	"errors"
	"fmt"

	"github.com/cilium/ebpf"
	"github.com/cilium/ebpf/link"
	"github.com/cilium/ebpf/ringbuf"
	"github.com/cilium/ebpf/rlimit"
)

// Tracer loads compiled eBPF programs into the kernel, attaches kprobes and
// tracepoints, and streams decoded events from ring buffers.
//
// Create with New, read events with ReadOpenat or ReadExec, read the vfs_read
// latency histogram with ReadLatencyHistogram, and call Close when done.
//
// Requires Linux 5.8+ and CAP_BPF+CAP_PERFMON (or root).
type Tracer struct {
	objs   tracerObjects // populated by loadTracerObjects (generated by bpf2go)
	links  []link.Link
	openRd *ringbuf.Reader
	execRd *ringbuf.Reader
}

// New loads the embedded eBPF object (generated by bpf2go via go generate),
// creates kernel maps, runs the verifier, and attaches probes.
// Returns ErrNotRoot if the process lacks the required capabilities.
func New() (*Tracer, error) {
	// RemoveMemlock removes the RLIMIT_MEMLOCK limit required on kernels < 5.11.
	// It is a no-op on 5.11+ where the BPF memory accounting was redesigned.
	if err := rlimit.RemoveMemlock(); err != nil {
		return nil, fmt.Errorf("%w: remove RLIMIT_MEMLOCK: %v", ErrNotRoot, err)
	}

	var objs tracerObjects
	if err := loadTracerObjects(&objs, nil); err != nil {
		var ve *ebpf.VerifierError
		if errors.As(err, &ve) {
			return nil, fmt.Errorf("tracer: BPF verifier rejected program:\n%v", ve)
		}
		return nil, fmt.Errorf("tracer: load BPF objects: %w", err)
	}

	t := &Tracer{objs: objs}

	// cleanup calls t.Close() and returns a typed error. Used to unwind
	// partially-initialized state on attach failures.
	cleanup := func(err error) (*Tracer, error) {
		t.Close()
		return nil, err
	}

	kpOpen, err := link.Kprobe("sys_openat", objs.KprobeOpenat, nil)
	if err != nil {
		return cleanup(fmt.Errorf("tracer: kprobe/sys_openat: %w", err))
	}
	t.links = append(t.links, kpOpen)

	kpVfsRead, err := link.Kprobe("vfs_read", objs.KprobeVfsRead, nil)
	if err != nil {
		return cleanup(fmt.Errorf("tracer: kprobe/vfs_read: %w", err))
	}
	t.links = append(t.links, kpVfsRead)

	krpVfsRead, err := link.Kretprobe("vfs_read", objs.KretprobeVfsRead, nil)
	if err != nil {
		return cleanup(fmt.Errorf("tracer: kretprobe/vfs_read: %w", err))
	}
	t.links = append(t.links, krpVfsRead)

	tpExec, err := link.Tracepoint("sched", "sched_process_exec", objs.TracepointSchedProcessExec, nil)
	if err != nil {
		return cleanup(fmt.Errorf("tracer: tracepoint/sched_process_exec: %w", err))
	}
	t.links = append(t.links, tpExec)

	t.openRd, err = ringbuf.NewReader(objs.OpenatEvents)
	if err != nil {
		return cleanup(fmt.Errorf("tracer: ring buffer (openat): %w", err))
	}

	t.execRd, err = ringbuf.NewReader(objs.ExecEvents)
	if err != nil {
		return cleanup(fmt.Errorf("tracer: ring buffer (exec): %w", err))
	}

	return t, nil
}

// ReadOpenat reads the next file-open event from the ring buffer.
// It blocks until an event arrives or Close is called.
// Returns ErrClosed when the tracer has been closed.
func (t *Tracer) ReadOpenat() (OpenatEvent, error) {
	rec, err := t.openRd.Read()
	if err != nil {
		if errors.Is(err, ringbuf.ErrClosed) {
			return OpenatEvent{}, ErrClosed
		}
		return OpenatEvent{}, fmt.Errorf("tracer: read openat event: %w", err)
	}
	return DecodeOpenat(rec.RawSample)
}

// ReadExec reads the next process-execution event from the ring buffer.
// It blocks until an event arrives or Close is called.
// Returns ErrClosed when the tracer has been closed.
func (t *Tracer) ReadExec() (ExecEvent, error) {
	rec, err := t.execRd.Read()
	if err != nil {
		if errors.Is(err, ringbuf.ErrClosed) {
			return ExecEvent{}, ErrClosed
		}
		return ExecEvent{}, fmt.Errorf("tracer: read exec event: %w", err)
	}
	return DecodeExec(rec.RawSample)
}

// ReadLatencyHistogram reads the current vfs_read latency histogram from the
// kernel BPF array map. Each bucket i contains the count of vfs_read calls
// with measured latency in [2^i, 2^(i+1)-1] nanoseconds.
//
// Call this before Close; the map is released when Close is called.
func (t *Tracer) ReadLatencyHistogram() (*Histogram, error) {
	var h Histogram
	for i := 0; i < histBuckets; i++ {
		key := uint32(i)
		var count uint64
		if err := t.objs.LatencyHist.Lookup(key, &count); err != nil {
			return nil, fmt.Errorf("tracer: latency histogram bucket %d: %w", i, err)
		}
		h.counts[i] = count
		h.total += count
	}
	return &h, nil
}

// Close detaches probes, closes ring buffer readers, and releases all kernel
// resources (maps, programs). It is safe to call Close more than once.
func (t *Tracer) Close() {
	if t.openRd != nil {
		t.openRd.Close()
	}
	if t.execRd != nil {
		t.execRd.Close()
	}
	for _, l := range t.links {
		l.Close()
	}
	t.objs.Close()
}
```

The cleanup closure pattern centralizes the partial-initialization teardown so each early-return does not repeat `kpOpen.Close(); objs.Close()` chains. Calling `t.Close()` on a partially-initialized `Tracer` is safe because every field is nil-checked or guarded by the slice being empty.

```go
// makeOpenatRaw serializes an OpenatEvent to its wire format.
// This helper constructs the bytes that the kernel ring buffer would produce.
func makeOpenatRaw(pid, uid uint32, tsNs uint64, filename string) []byte {
	var buf bytes.Buffer
	_ = binary.Write(&buf, binary.LittleEndian, pid)
	_ = binary.Write(&buf, binary.LittleEndian, uid)
	_ = binary.Write(&buf, binary.LittleEndian, tsNs)
	var fn [256]byte
	copy(fn[:], filename)
	buf.Write(fn[:])
	return buf.Bytes()
}

// makeExecRaw serializes an ExecEvent to its wire format.
func makeExecRaw(pid, ppid uint32, tsNs uint64, comm, filename string) []byte {
	var buf bytes.Buffer
	_ = binary.Write(&buf, binary.LittleEndian, pid)
	_ = binary.Write(&buf, binary.LittleEndian, ppid)
	_ = binary.Write(&buf, binary.LittleEndian, tsNs)
	var c [16]byte
	copy(c[:], comm)
	buf.Write(c[:])
	var fn [256]byte
	copy(fn[:], filename)
	buf.Write(fn[:])
	return buf.Bytes()
}
```

```go
// runOpenat streams file-open events until Ctrl-C.
// Close unblocks the blocked ReadOpenat call so the reader goroutine exits cleanly.
func runOpenat(ctx context.Context, t *tracer.Tracer) {
	fmt.Println("tracing openat(2) system calls... Ctrl-C to stop")
	done := make(chan struct{})
	go func() {
		defer close(done)
		for {
			ev, err := t.ReadOpenat()
			if errors.Is(err, tracer.ErrClosed) {
				return
			}
			if err != nil {
				log.Printf("ReadOpenat: %v", err)
				continue
			}
			fmt.Printf("PID=%-6d UID=%-6d %s\n", ev.PID, ev.UID, ev.FilenameStr())
		}
	}()
	<-ctx.Done()
	t.Close()
	<-done
}

// runExec streams process-execution events until Ctrl-C.
func runExec(ctx context.Context, t *tracer.Tracer) {
	fmt.Println("tracing exec... Ctrl-C to stop")
	done := make(chan struct{})
	go func() {
		defer close(done)
		for {
			ev, err := t.ReadExec()
			if errors.Is(err, tracer.ErrClosed) {
				return
			}
			if err != nil {
				log.Printf("ReadExec: %v", err)
				continue
			}
			fmt.Printf("PID=%-6d PPID=%-6d %-16s %s\n",
				ev.PID, ev.PPID, ev.CommStr(), ev.FilenameStr())
		}
	}()
	<-ctx.Done()
	t.Close()
	<-done
}

// runHistogram samples vfs_read latency for 5 seconds, then prints the histogram.
// ReadLatencyHistogram is called before Close because the BPF map is released by Close.
func runHistogram(ctx context.Context, t *tracer.Tracer) {
	defer t.Close()
	fmt.Println("sampling vfs_read latency for 5 s (Ctrl-C to sample early)...")
	select {
	case <-time.After(5 * time.Second):
	case <-ctx.Done():
	}
	h, err := t.ReadLatencyHistogram()
	if err != nil {
		log.Fatalf("ReadLatencyHistogram: %v", err)
	}
	fmt.Printf("vfs_read latency (%d samples):\n%s", h.Total(), h)
}
```

## Common Mistakes

### Struct Layout Mismatch Silently Corrupts Fields

Wrong: defining the Go struct without accounting for C alignment padding:

```go
// Wrong: __u32 pid followed immediately by __u64 ts_ns in C has 4 bytes of
// padding inserted between them; this Go struct omits the padding.
type BadEvent struct {
	PID  uint32
	TsNs uint64
	// ...
}
```

What happens: `binary.Read` fills PID correctly, then reads the next 8 bytes as TsNs — but those bytes include the 4 bytes of C padding plus the first 4 bytes of the actual `ts_ns` value. Every field after PID is wrong, and there is no compile error or runtime panic.

Fix: design C structs to avoid padding by using pairs of `__u32` before each `__u64` (as in `openat_event` and `exec_event` above), or add an explicit padding field on both sides. Always add a size assertion comment with the expected byte count.

### Calling Close Before ReadLatencyHistogram

Wrong: closing the tracer before reading the histogram map:

```go
defer t.Close() // this releases the BPF maps
h, err := t.ReadLatencyHistogram() // the map is now invalid
```

What happens: `LatencyHist.Lookup` returns an error like "use of closed file descriptor", and the histogram is lost.

Fix: read the histogram before closing. In `runHistogram`, `ReadLatencyHistogram()` runs inside the function body before `defer t.Close()` fires.

### Ignoring the BPF Verifier Error

Wrong: treating all `loadTracerObjects` errors as generic failures:

```go
if err := loadTracerObjects(&objs, nil); err != nil {
	return nil, fmt.Errorf("load: %w", err)
}
```

What happens: when the kernel verifier rejects a program, the error message contains the full verifier log with the instruction that violated the constraint. Wrapping it as a generic error loses that diagnostic.

Fix: check for `*ebpf.VerifierError` with `errors.As` and print the full error. The verifier log tells you which instruction caused the rejection and which constraint it violated.

### Not Unblocking ReadOpenat Before Calling Close

Wrong: calling `t.Close()` from the same goroutine that is blocking on `t.ReadOpenat()`:

```go
// Deadlock: this goroutine is blocked in ReadOpenat and never reaches Close.
ev, _ := t.ReadOpenat()
t.Close()
```

What happens: the goroutine blocks forever because `ReadOpenat` never returns.

Fix: close the tracer from a separate goroutine (as in `runOpenat` above). `Close` closes the ring buffer reader, which unblocks the blocked `Read` call and causes it to return `ringbuf.ErrClosed`, which `ReadOpenat` translates to `ErrClosed`.

## Summary

- eBPF programs are compiled C that the kernel verifier proves safe before JIT-compiling to native code; they attach to kprobes, kretprobes, tracepoints, uprobes, and XDP hook points.
- `BPF_MAP_TYPE_RINGBUF` is the modern kernel-to-userspace channel: lock-free, ordered, no per-CPU fragmentation.
- Go structs that mirror C event structs must match byte-for-byte including alignment padding; two `__u32` fields before a `__u64` avoid padding entirely.
- `bpf2go` compiles the C source and generates Go bindings with embedded BPF bytecode, producing self-contained binaries that need no external `.o` files.
- `log2Bucket` using `bits.LeadingZeros64` maps a nanosecond latency to a 64-bucket histogram in O(1); 64 buckets cover 1 ns to ~580 years.
- `ReadOpenat` and `ReadExec` block until an event arrives; calling `Close` from another goroutine unblocks them by closing the underlying ring buffer reader.
- `ReadLatencyHistogram` must be called before `Close`; the BPF array map is released when Close runs.
