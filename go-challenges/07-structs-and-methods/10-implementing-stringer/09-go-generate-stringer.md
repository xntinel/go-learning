# Exercise 9: go:generate With golang.org/x/tools/cmd/stringer For A Large Enum

A fifteen-value enum with a hand-written `switch` is a maintenance liability: add a
constant, forget an arm, and `String()` silently returns the wrong name. This
module replaces the switch with `//go:generate stringer -type=ErrorCode`, studies
the `_name`/`_index` arrays the tool emits, and adds a CI guard test that fails if
a constant is added without regenerating.

Self-contained module: own `go mod init`, code, demo, and tests. The generated
file is committed so the module builds offline without running the tool.

## What you'll build

```text
errcode/                    independent module: example.com/errcode
  go.mod
  errcode.go                type ErrorCode; 17 gRPC-style constants; //go:generate; allCodes
  errcode_string.go         committed stringer output: _name/_index String()
  cmd/
    demo/
      main.go               prints every code and an out-of-range value
  errcode_test.go           names non-empty/unique; ErrorCode(N) fallback; drift guard
```

- Files: `errcode.go`, `errcode_string.go`, `cmd/demo/main.go`, `errcode_test.go`.
- Implement: a 17-value `ErrorCode` with a `//go:generate stringer` directive and a maintained `allCodes` slice; the committed generated `String()` backed by `_name`/`_index`.
- Test: every declared code maps to a non-empty unique name; an out-of-range value renders `ErrorCode(N)`; a drift guard that fails when a constant is missing from the generated table.
- Verify: `go test -count=1 -race ./...`

### What stringer generates and why it is fast

The tool `golang.org/x/tools/cmd/stringer` reads the constants of a type and emits
a `String()` method backed by two package-level values: a single concatenated
`_ErrorCode_name` string ("OKCanceledUnknown...") and an `_ErrorCode_index` array
of offsets into it. `String()` for value `i` returns
`_name[_index[i]:_index[i+1]]` — one slice of a static string, no allocation, no
map, no switch. For a contiguous run of constants (values `0..N` with no gaps, as
here) that is all it needs; for gapped or multi-run enums it emits several
`_name`/`_index` pairs and a small dispatch, but the shape is the same.

Two properties of the generated file matter in practice. First, it includes a
compile-time guard function `func _()` full of `_ = x[OK-0]` lines: if you change a
constant's *value* without regenerating, one of those array indices goes negative
or out of range and the package *fails to compile* — a loud, immediate signal.
Second — and this is the trap — that guard does **not** fire if you *insert* a new
constant in the middle (shifting later values) but the compile-time checks still
line up, or if you *append* a constant without rerunning the tool: the new value
simply falls past the end of `_index` and renders as the `ErrorCode(N)` fallback.
Appending is the common case, and it is silent, so a CI guard test that fails when
a declared constant has no generated name is worth its weight.

The out-of-range fallback the tool emits is exactly the `TypeName(N)` convention:
`ErrorCode(99)`. That is why an appended-but-not-regenerated constant is diagnosable
rather than blank.

Create `errcode.go`. The `//go:generate` directive documents how the sibling file
was produced; `allCodes` is the maintained list the drift guard walks:

```go
package errcode

//go:generate stringer -type=ErrorCode

// ErrorCode is a gRPC-style canonical status code. String() is generated by
// stringer into errcode_string.go; do not hand-write it.
type ErrorCode int

const (
	OK ErrorCode = iota
	Canceled
	Unknown
	InvalidArgument
	DeadlineExceeded
	NotFound
	AlreadyExists
	PermissionDenied
	ResourceExhausted
	FailedPrecondition
	Aborted
	OutOfRange
	Unimplemented
	Internal
	Unavailable
	DataLoss
	Unauthenticated
)

// allCodes lists every declared code in order. The drift-guard test walks it, so
// a constant appended above without rerunning `go generate` is caught in CI.
var allCodes = []ErrorCode{
	OK, Canceled, Unknown, InvalidArgument, DeadlineExceeded,
	NotFound, AlreadyExists, PermissionDenied, ResourceExhausted,
	FailedPrecondition, Aborted, OutOfRange, Unimplemented,
	Internal, Unavailable, DataLoss, Unauthenticated,
}
```

Now the committed generated file. This is verbatim what
`stringer -type=ErrorCode` produces for the constants above — the compile-time
guard, the concatenated name string, the offset index, and the slicing `String()`.

Create `errcode_string.go`:

```go
// Code generated by "stringer -type=ErrorCode"; DO NOT EDIT.

package errcode

import "strconv"

func _() {
	// An "invalid array index" compiler error signifies that the constant values have changed.
	// Re-run the stringer command to generate them again.
	var x [1]struct{}
	_ = x[OK-0]
	_ = x[Canceled-1]
	_ = x[Unknown-2]
	_ = x[InvalidArgument-3]
	_ = x[DeadlineExceeded-4]
	_ = x[NotFound-5]
	_ = x[AlreadyExists-6]
	_ = x[PermissionDenied-7]
	_ = x[ResourceExhausted-8]
	_ = x[FailedPrecondition-9]
	_ = x[Aborted-10]
	_ = x[OutOfRange-11]
	_ = x[Unimplemented-12]
	_ = x[Internal-13]
	_ = x[Unavailable-14]
	_ = x[DataLoss-15]
	_ = x[Unauthenticated-16]
}

const _ErrorCode_name = "OKCanceledUnknownInvalidArgumentDeadlineExceededNotFoundAlreadyExistsPermissionDeniedResourceExhaustedFailedPreconditionAbortedOutOfRangeUnimplementedInternalUnavailableDataLossUnauthenticated"

var _ErrorCode_index = [...]uint8{0, 2, 10, 17, 32, 48, 56, 69, 85, 102, 120, 127, 137, 150, 158, 169, 177, 192}

func (i ErrorCode) String() string {
	if i < 0 || i >= ErrorCode(len(_ErrorCode_index)-1) {
		return "ErrorCode(" + strconv.FormatInt(int64(i), 10) + ")"
	}
	return _ErrorCode_name[_ErrorCode_index[i]:_ErrorCode_index[i+1]]
}
```

### The runnable demo

The demo prints every code with its integer value and generated name, then an
out-of-range value to show the fallback.

Create `cmd/demo/main.go`:

```go
package main

import (
	"fmt"

	"example.com/errcode"
)

func main() {
	for _, c := range []errcode.ErrorCode{
		errcode.OK, errcode.NotFound, errcode.PermissionDenied,
		errcode.Unavailable, errcode.Unauthenticated,
	} {
		fmt.Printf("%2d %s\n", int(c), c)
	}
	fmt.Printf("out of range: %s\n", errcode.ErrorCode(99))
}
```

Run it:

```bash
go run ./cmd/demo
```

Expected output:

```
 0 OK
 5 NotFound
 7 PermissionDenied
14 Unavailable
16 Unauthenticated
out of range: ErrorCode(99)
```

### Tests

`TestNamesNonEmptyUnique` walks `allCodes` and asserts each generated name is
non-empty, is not the numeric fallback, and is unique. `TestOutOfRange` pins the
`ErrorCode(N)` fallback for values past the table. `TestNoDrift` is the CI guard:
it asserts `allCodes` is contiguous from zero and that the generated index table
has exactly as many names as there are declared codes — so appending a constant
without rerunning `go generate` fails the build's tests, not production.

Create `errcode_test.go`:

```go
package errcode

import (
	"strconv"
	"strings"
	"testing"
)

func TestNamesNonEmptyUnique(t *testing.T) {
	t.Parallel()
	seen := make(map[string]bool, len(allCodes))
	for _, c := range allCodes {
		name := c.String()
		if name == "" {
			t.Errorf("code %d has empty name", int(c))
		}
		if strings.HasPrefix(name, "ErrorCode(") {
			t.Errorf("code %d fell through to numeric fallback %q; regenerate stringer", int(c), name)
		}
		if seen[name] {
			t.Errorf("duplicate name %q", name)
		}
		seen[name] = true
	}
}

func TestExpectedNames(t *testing.T) {
	t.Parallel()
	want := map[ErrorCode]string{
		OK:               "OK",
		NotFound:         "NotFound",
		PermissionDenied: "PermissionDenied",
		Internal:         "Internal",
		Unauthenticated:  "Unauthenticated",
	}
	for c, w := range want {
		if got := c.String(); got != w {
			t.Errorf("ErrorCode(%d).String() = %q, want %q", int(c), got, w)
		}
	}
}

func TestOutOfRange(t *testing.T) {
	t.Parallel()
	for _, c := range []ErrorCode{99, -1, 17} {
		want := "ErrorCode(" + strconv.Itoa(int(c)) + ")"
		if got := c.String(); got != want {
			t.Errorf("ErrorCode(%d).String() = %q, want %q", int(c), got, want)
		}
	}
}

func TestNoDrift(t *testing.T) {
	t.Parallel()
	for i, c := range allCodes {
		if int(c) != i {
			t.Fatalf("allCodes not contiguous: index %d holds value %d", i, int(c))
		}
	}
	if names := len(_ErrorCode_index) - 1; names != len(allCodes) {
		t.Fatalf("generated table names %d codes but %d are declared; run `go generate`", names, len(allCodes))
	}
}
```

## Review

The generated `String()` is fast because it slices a static string through an
offset table — no allocation, no switch — which is exactly why you want the tool
rather than a hand-written method for a large enum. The subtlety is drift: the
compile-time guard function catches *changed values* but not an *appended*
constant, and appending is the usual mistake, so `TestNoDrift` and the
non-empty/unique check are the safety net that turns a silent wrong name into a
failing test. Keep the generated file committed and regenerate with `go generate
./...` whenever the constant set changes; treat `errcode_string.go` as build
output, never edit it by hand. The `ErrorCode(N)` fallback means even a drifted
value is diagnosable in a log rather than blank.
