# Exercise 1: Transcode a gRPC service to REST with google.api.http annotations

You own an `OrderService` gRPC API and product needs a REST/JSON surface for it. Rather
than write a second HTTP service, you annotate the existing `.proto` with
`google.api.http` rules and let grpc-gateway generate a reverse proxy. This exercise is
the real on-the-job task: cover every method-to-route case a CRUD API needs, then wire
the generated proxy in-process (no socket) with a dialed variant behind a build tag.

## What you'll build

```text
gateway/                         module: example.com/gateway
  go.mod
  proto/order/v1/order.proto     annotated OrderService (get/list/post/patch/delete + additional_bindings)
  gen/order/v1/                  generated by buf/protoc (orderpb: messages + Register*Handler* funcs)
  gateway.go                     NewInProcessMux: runtime.NewServeMux + RegisterOrderServiceHandlerServer
  gateway_remote.go             //go:build remotegw: NewDialedMux with grpc.NewClient + RegisterOrderServiceHandler
  cmd/demo/main.go               in-memory server behind an httptest server, two routes to one RPC
  gateway_test.go                httptest against the mux: routing, additional_bindings, query params, body:"order"
```

Files: `proto/order/v1/order.proto`, `gateway.go`, `gateway_remote.go`, `cmd/demo/main.go`, `gateway_test.go`.
Implement: an annotated `OrderService`, an in-process mux via `RegisterOrderServiceHandlerServer`, and a dialed mux via `grpc.NewClient` + `RegisterOrderServiceHandler`.
Test: drive the mux with `httptest` + `http.Client`, asserting path params, `additional_bindings`, ignored query params, and `body:"order"` sub-message binding.
Verify: `buf generate && go test -count=1 -race ./...` (codegen produces `gen/order/v1`; offline, without the plugins and modules, build fails by design — this is a bar-mode lesson).

Set up the module:

```bash
mkdir -p ~/go-exercises/gateway/proto/order/v1 ~/go-exercises/gateway/cmd/demo
cd ~/go-exercises/gateway
go mod init example.com/gateway
go get github.com/grpc-ecosystem/grpc-gateway/v2/runtime
go get google.golang.org/grpc google.golang.org/protobuf
```

### The annotation grammar, one RPC at a time

The `.proto` is authoritative for both surfaces. Each RPC carries an
`option (google.api.http)` that a single line turns into an HTTP route. The dependency
that makes this compile is `import "google/api/annotations.proto"` — the googleapis
protos must be on your `buf`/`protoc` include path or codegen silently emits no gateway.

Read the mapping as you go:

- `GetOrder` uses `get: "/v1/orders/{id}"`. `{id}` captures one path segment into the
  request's `id` field. It also declares `additional_bindings` with a legacy
  `get: "/orders/{id}"` — both routes resolve to the *same* handler, which is how you keep
  an old path alive without duplicating logic.
- `ListOrders` uses `get: "/v1/orders"` with no captures, so its request fields
  (`page_size`, `state`) become query parameters. Unknown query parameters are ignored, so
  a client sending a stale param does not get a 400.
- `CreateOrder` uses `post: "/v1/orders"` with `body: "*"`, binding the entire JSON body
  onto the request message.
- `UpdateOrder` uses `patch: "/v1/orders/{order.id}"` with `body: "order"`. The path
  captures the nested `order.id`; `body: "order"` binds the JSON body onto *only* the
  `order` sub-message, leaving `update_mask` (a separate field) to be a query parameter.
- `DeleteOrder` uses `delete: "/v1/orders/{id}"` and returns `google.protobuf.Empty`.

Create `proto/order/v1/order.proto`:

```proto
syntax = "proto3";

package order.v1;

import "google/api/annotations.proto";
import "google/protobuf/empty.proto";
import "google/protobuf/field_mask.proto";

option go_package = "example.com/gateway/gen/order/v1;orderpb";

service OrderService {
  rpc GetOrder(GetOrderRequest) returns (Order) {
    option (google.api.http) = {
      get: "/v1/orders/{id}"
      additional_bindings {get: "/orders/{id}"}
    };
  }

  rpc ListOrders(ListOrdersRequest) returns (ListOrdersResponse) {
    option (google.api.http) = {get: "/v1/orders"};
  }

  rpc CreateOrder(CreateOrderRequest) returns (Order) {
    option (google.api.http) = {
      post: "/v1/orders"
      body: "*"
    };
  }

  rpc UpdateOrder(UpdateOrderRequest) returns (Order) {
    option (google.api.http) = {
      patch: "/v1/orders/{order.id}"
      body: "order"
    };
  }

  rpc DeleteOrder(DeleteOrderRequest) returns (google.protobuf.Empty) {
    option (google.api.http) = {delete: "/v1/orders/{id}"};
  }
}

message Order {
  string id = 1;
  string customer = 2;
  int64 amount_cents = 3;
  string state = 4;
}

message GetOrderRequest {
  string id = 1;
}

message ListOrdersRequest {
  int32 page_size = 1;
  string state = 2;
}

message ListOrdersResponse {
  repeated Order orders = 1;
}

message CreateOrderRequest {
  string customer = 1;
  int64 amount_cents = 2;
}

message UpdateOrderRequest {
  Order order = 1;
  google.protobuf.FieldMask update_mask = 2;
}

message DeleteOrderRequest {
  string id = 1;
}
```

Generating with `buf generate` (using `protoc-gen-go`, `protoc-gen-go-grpc`, and
`protoc-gen-grpc-gateway`) produces the `orderpb` package in `gen/order/v1`: the message
types, the `OrderServiceServer` interface, `UnimplementedOrderServiceServer`, and the
gateway's `Register*Handler*` functions used below.

### In-process registration: no socket

The whole point of in-process registration is that the gateway calls your Go methods
directly. `RegisterOrderServiceHandlerServer(ctx, mux, srv)` takes the concrete server
value and wires each annotated route straight to it — nothing is serialized to the wire.
This is the right default for a single binary that serves both gRPC and REST, and it is
exactly what the tests use.

Create `gateway.go`:

```go
package gateway

import (
	"context"

	"github.com/grpc-ecosystem/grpc-gateway/v2/runtime"

	orderpb "example.com/gateway/gen/order/v1"
)

// NewInProcessMux builds a REST/JSON gateway that invokes srv directly. There is
// no socket between the HTTP layer and the gRPC handler: one process, one failure
// domain, zero network hops. This is the right default for a single binary.
func NewInProcessMux(ctx context.Context, srv orderpb.OrderServiceServer) (*runtime.ServeMux, error) {
	mux := runtime.NewServeMux()
	if err := orderpb.RegisterOrderServiceHandlerServer(ctx, mux, srv); err != nil {
		return nil, err
	}
	return mux, nil
}
```

### The dialed variant, behind a build tag

When the gRPC service really is a separate process, the gateway dials it. The current
constructor is `grpc.NewClient(target, opts...)` — `grpc.DialContext`/`grpc.Dial` with
`WithBlock` are deprecated. A plaintext dev endpoint needs
`grpc.WithTransportCredentials(insecure.NewCredentials())`, or the dial fails on transport
security. `RegisterOrderServiceHandler(ctx, mux, conn)` registers the routes against the
client connection. (`RegisterOrderServiceHandlerFromEndpoint(ctx, mux, endpoint, opts)` is
the convenience wrapper that builds the connection for you.) This lives behind a build tag
so the default build stays in-process; use it only when the hop is real.

Create `gateway_remote.go`:

```go
//go:build remotegw

package gateway

import (
	"context"

	"github.com/grpc-ecosystem/grpc-gateway/v2/runtime"
	"google.golang.org/grpc"
	"google.golang.org/grpc/credentials/insecure"

	orderpb "example.com/gateway/gen/order/v1"
)

// NewDialedMux builds a gateway that dials a standalone gRPC endpoint. Use it only
// when the service runs in another process: it adds a network hop, its own timeouts,
// and a second failure domain. In the same binary, prefer NewInProcessMux.
func NewDialedMux(ctx context.Context, endpoint string) (*runtime.ServeMux, error) {
	conn, err := grpc.NewClient(
		endpoint,
		grpc.WithTransportCredentials(insecure.NewCredentials()),
	)
	if err != nil {
		return nil, err
	}
	mux := runtime.NewServeMux()
	if err := orderpb.RegisterOrderServiceHandler(ctx, mux, conn); err != nil {
		return nil, err
	}
	return mux, nil
}
```

### The runnable demo

The demo wires an in-memory `OrderService` in-process, wraps the mux in an
`httptest.Server`, and issues the two `GetOrder` routes so you can see both the `/v1` path
and the legacy path resolve to one handler. The demo sets no marshaler option, so the JSON
is grpc-gateway's default `protojson` shape: lowerCamelCase field names, and — a real
protojson detail — a 64-bit integer (`amount_cents`) is encoded as a JSON *string*.

One more protojson detail matters for a reproducible demo: protojson's single-line
encoder deliberately inserts a random extra space after *some* commas (a `detrand`
anti-stability measure), so the raw bytes differ run to run. The demo therefore runs the
body through `json.Compact` before printing, which strips all insignificant whitespace and
yields a stable, comparable line. Never depend on protojson's byte layout being stable;
normalize before you compare or print.

Create `cmd/demo/main.go`:

```go
package main

import (
	"bytes"
	"context"
	"encoding/json"
	"fmt"
	"io"
	"log"
	"net/http"
	"net/http/httptest"

	"example.com/gateway"
	orderpb "example.com/gateway/gen/order/v1"
)

// memServer is a tiny in-memory OrderService used to demonstrate transcoding.
type memServer struct {
	orderpb.UnimplementedOrderServiceServer
}

func (memServer) GetOrder(_ context.Context, req *orderpb.GetOrderRequest) (*orderpb.Order, error) {
	return &orderpb.Order{
		Id:          req.GetId(),
		Customer:    "alice",
		AmountCents: 4200,
		State:       "OPEN",
	}, nil
}

func main() {
	mux, err := gateway.NewInProcessMux(context.Background(), memServer{})
	if err != nil {
		log.Fatal(err)
	}
	srv := httptest.NewServer(mux)
	defer srv.Close()

	for _, path := range []string{"/v1/orders/o-123", "/orders/o-123"} {
		resp, err := http.Get(srv.URL + path)
		if err != nil {
			log.Fatal(err)
		}
		raw, _ := io.ReadAll(resp.Body)
		resp.Body.Close()
		// protojson may insert a nondeterministic space after commas; compact it
		// so the printed line is stable across runs.
		var body bytes.Buffer
		if err := json.Compact(&body, raw); err != nil {
			log.Fatal(err)
		}
		fmt.Printf("GET %s -> %d %s\n", path, resp.StatusCode, body.Bytes())
	}
}
```

Run it:

```bash
go run ./cmd/demo
```

Expected output:

```
GET /v1/orders/o-123 -> 200 {"id":"o-123","customer":"alice","amountCents":"4200","state":"OPEN"}
GET /orders/o-123 -> 200 {"id":"o-123","customer":"alice","amountCents":"4200","state":"OPEN"}
```

### Testing transcoding without a live gRPC socket

The pattern that lets you test transcoding with no gRPC server on a port: register a
concrete fake in-process with `RegisterOrderServiceHandlerServer`, wrap the mux in
`httptest.NewServer`, and drive it with an `http.Client`. The fake records what it received
so each test can assert that the route mapped to the right RPC with the right fields. The
cases prove the four mapping rules that matter: a path parameter reaches `GetOrder`; the two
`additional_bindings` paths both hit the one handler; an unknown query parameter is ignored;
and `body: "order"` unmarshals *only* the `order` sub-message on `UpdateOrder`.

Create `gateway_test.go`:

```go
package gateway_test

import (
	"bytes"
	"context"
	"fmt"
	"io"
	"net/http"
	"net/http/httptest"
	"sync"
	"testing"

	"example.com/gateway"
	orderpb "example.com/gateway/gen/order/v1"
	"google.golang.org/protobuf/types/known/emptypb"
)

// recordingServer captures the last request each RPC saw so tests can assert how
// the gateway populated the message from path, body, and query.
type recordingServer struct {
	orderpb.UnimplementedOrderServiceServer

	mu         sync.Mutex
	lastGetID  string
	listCalls  int
	lastUpdate *orderpb.UpdateOrderRequest
}

func (s *recordingServer) GetOrder(_ context.Context, req *orderpb.GetOrderRequest) (*orderpb.Order, error) {
	s.mu.Lock()
	s.lastGetID = req.GetId()
	s.mu.Unlock()
	return &orderpb.Order{Id: req.GetId(), State: "OPEN"}, nil
}

func (s *recordingServer) ListOrders(_ context.Context, _ *orderpb.ListOrdersRequest) (*orderpb.ListOrdersResponse, error) {
	s.mu.Lock()
	s.listCalls++
	s.mu.Unlock()
	return &orderpb.ListOrdersResponse{}, nil
}

func (s *recordingServer) UpdateOrder(_ context.Context, req *orderpb.UpdateOrderRequest) (*orderpb.Order, error) {
	s.mu.Lock()
	s.lastUpdate = req
	s.mu.Unlock()
	return req.GetOrder(), nil
}

func (s *recordingServer) DeleteOrder(_ context.Context, _ *orderpb.DeleteOrderRequest) (*emptypb.Empty, error) {
	return &emptypb.Empty{}, nil
}

func newTestServer(t *testing.T) (*httptest.Server, *recordingServer) {
	t.Helper()
	fake := &recordingServer{}
	mux, err := gateway.NewInProcessMux(context.Background(), fake)
	if err != nil {
		t.Fatalf("NewInProcessMux: %v", err)
	}
	srv := httptest.NewServer(mux)
	t.Cleanup(srv.Close)
	return srv, fake
}

func TestPathAndBindings(t *testing.T) {
	t.Parallel()

	tests := []struct {
		name   string
		path   string
		wantID string
	}{
		{name: "v1 route", path: "/v1/orders/o-1", wantID: "o-1"},
		{name: "legacy additional_binding", path: "/orders/o-2", wantID: "o-2"},
	}
	for _, tt := range tests {
		t.Run(tt.name, func(t *testing.T) {
			srv, fake := newTestServer(t)

			resp, err := http.Get(srv.URL + tt.path)
			if err != nil {
				t.Fatalf("GET %s: %v", tt.path, err)
			}
			io.Copy(io.Discard, resp.Body)
			resp.Body.Close()

			if resp.StatusCode != http.StatusOK {
				t.Fatalf("status = %d; want 200", resp.StatusCode)
			}
			fake.mu.Lock()
			got := fake.lastGetID
			fake.mu.Unlock()
			if got != tt.wantID {
				t.Fatalf("GetOrder id = %q; want %q", got, tt.wantID)
			}
		})
	}
}

func TestUnknownQueryParamIgnored(t *testing.T) {
	t.Parallel()
	srv, fake := newTestServer(t)

	resp, err := http.Get(srv.URL + "/v1/orders?page_size=50&bogus=1")
	if err != nil {
		t.Fatalf("GET list: %v", err)
	}
	io.Copy(io.Discard, resp.Body)
	resp.Body.Close()

	if resp.StatusCode != http.StatusOK {
		t.Fatalf("status = %d; want 200 (unknown query params must be ignored)", resp.StatusCode)
	}
	fake.mu.Lock()
	calls := fake.listCalls
	fake.mu.Unlock()
	if calls != 1 {
		t.Fatalf("ListOrders calls = %d; want 1", calls)
	}
}

func TestPatchBodyBindsSubMessageOnly(t *testing.T) {
	t.Parallel()
	srv, fake := newTestServer(t)

	body := `{"customer":"bob","amount_cents":"999"}`
	req, err := http.NewRequest(http.MethodPatch, srv.URL+"/v1/orders/o-7", bytes.NewBufferString(body))
	if err != nil {
		t.Fatalf("new request: %v", err)
	}
	req.Header.Set("Content-Type", "application/json")

	resp, err := http.DefaultClient.Do(req)
	if err != nil {
		t.Fatalf("PATCH: %v", err)
	}
	io.Copy(io.Discard, resp.Body)
	resp.Body.Close()

	if resp.StatusCode != http.StatusOK {
		t.Fatalf("status = %d; want 200", resp.StatusCode)
	}
	fake.mu.Lock()
	upd := fake.lastUpdate
	fake.mu.Unlock()
	if upd == nil {
		t.Fatal("UpdateOrder was not called")
	}
	// The path segment {order.id} must populate order.id; body:"order" binds the
	// JSON onto the order sub-message.
	if got := upd.GetOrder().GetId(); got != "o-7" {
		t.Fatalf("order.id = %q; want o-7 (from path)", got)
	}
	if got := upd.GetOrder().GetCustomer(); got != "bob" {
		t.Fatalf("order.customer = %q; want bob (from body)", got)
	}
}

func ExampleNewInProcessMux() {
	mux, _ := gateway.NewInProcessMux(context.Background(), &recordingServer{})
	srv := httptest.NewServer(mux)
	defer srv.Close()

	resp, _ := http.Get(srv.URL + "/v1/orders/o-42")
	body, _ := io.ReadAll(resp.Body)
	resp.Body.Close()
	fmt.Printf("%d\n", resp.StatusCode)
	_ = body
	// Output: 200
}
```

## Review

The gateway is correct when every route resolves to the intended RPC with the request
message populated from the right source. The tests pin that: `TestPathAndBindings` proves
`{id}` reaches `GetOrder` and that the legacy `additional_bindings` path hits the same
handler; `TestUnknownQueryParamIgnored` proves a stray query parameter does not 400;
`TestPatchBodyBindsSubMessageOnly` proves `{order.id}` comes from the path while
`body: "order"` fills the `order` sub-message. If a route 404s, the annotation's verb/path
is wrong or the annotation imports were missing at codegen. If `PATCH` unmarshals the whole
request instead of the sub-message, `body` was set to `"*"` instead of `"order"`.

The mistakes to avoid are structural. Do not reach for `RegisterOrderServiceHandlerFromEndpoint`
to dial `localhost` when the server is in this binary — that adds serialization and a
loopback hop for nothing; `RegisterOrderServiceHandlerServer` calls the methods directly.
Do not forget `google/api/annotations.proto` in your `buf` deps or `protoc` include path, or
codegen emits no gateway. And remember this is a bar-mode lesson: the wiring depends on the
`buf`/`protoc`-generated `orderpb` package, so it compiles only after `buf generate`, not in
the offline gate.
