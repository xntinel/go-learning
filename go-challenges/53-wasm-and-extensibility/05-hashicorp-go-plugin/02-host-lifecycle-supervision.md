# Exercise 2: Host-Side Plugin Lifecycle and Supervision

This is the on-the-job piece: treating a plugin as a supervised child process.
You build the host that launches, connects to, health-checks, and tears down a
plugin — negotiating the handshake, dispensing the typed interface, pinging for
liveness, bounding startup with a timeout, detecting a crashed plugin, and reaping
orphans with `CleanupClients`.

This module is fully self-contained: it bundles its own copy of the contract from
Exercise 1 (nothing is imported across exercises) plus the host supervisor and the
plugin binary. It is bar-mode — it depends on `hashicorp/go-plugin`, `hclog`,
`grpc`, and a generated proto package, so it does not build offline; the value is
the correct supervision shape, verified APIs, and a deterministic reattach test.


Files: `shared/processor.go`, `host/host.go`, `host/host_test.go`, `host/host_integration_test.go`, `plugin/main.go`, `cmd/demo/main.go`.
Implement: a `Manager` that wraps `plugin.NewClient` with a `Cmd`, connects with `Client()`, `Dispense`s the typed `Processor`, health-checks with `Ping`, reports `Exited`/`NegotiatedVersion`, and tears down with `Kill`; a sentinel `ErrWrongType`.
Test: over the reattach harness assert `Dispense` returns the interface and a call succeeds, `Ping` is nil while alive and non-nil after teardown, and the type-assertion path yields `ErrWrongType` via `errors.Is`; a `//go:build manual` integration test drives the real binary and a too-short `StartTimeout`.
Verify: `go test -race ./...` offline (reattach path) and `go test -tags manual ./...` against a built binary, with deps present.

### The contract, bundled

The shared contract is identical in shape to Exercise 1: the `Processor`
interface, the `Handshake`, the `PluginMap`, and the `ProcessorGRPCPlugin` bridge.
It is repeated here so the module builds on its own. The proto (`Record`,
`ProcessRequest`, `ProcessResponse`, `Processor` service) is generated by the
`protoc` command above.

Create `shared/processor.go`:

```go
package shared

import (
	"context"
	"errors"
	"fmt"

	plugin "github.com/hashicorp/go-plugin"
	"google.golang.org/grpc"

	pb "example.com/pluginhost/proto"
)

// ErrEmptyBatch is returned by Validate when a request carries no records.
var ErrEmptyBatch = errors.New("processor: empty batch")

// Record is a domain record handed to the plugin.
type Record struct {
	ID   string
	Body string
}

// ProcessRequest is a coarse-grained batch, sized to amortize the process boundary.
type ProcessRequest struct {
	Records []Record
}

// Validate rejects an empty batch, wrapping ErrEmptyBatch with %w.
func (r ProcessRequest) Validate() error {
	if len(r.Records) == 0 {
		return fmt.Errorf("validate request: %w", ErrEmptyBatch)
	}
	return nil
}

// ProcessResponse carries transformed records and a processed count.
type ProcessResponse struct {
	Records   []Record
	Processed int64
}

// Processor is the domain interface host and plugin share.
type Processor interface {
	Process(ctx context.Context, req ProcessRequest) (ProcessResponse, error)
}

// Handshake gates the connection: magic cookie plus ABI version.
var Handshake = plugin.HandshakeConfig{
	ProtocolVersion:  1,
	MagicCookieKey:   "PROCESSOR_PLUGIN",
	MagicCookieValue: "8f2a1c7e-processor-v1",
}

// PluginMap is the dispensable plugin set.
var PluginMap = plugin.PluginSet{
	"processor": &ProcessorGRPCPlugin{},
}

// ProcessorGRPCPlugin bridges Processor to gRPC. The embedded
// NetRPCUnsupportedPlugin satisfies the base plugin.Plugin interface.
type ProcessorGRPCPlugin struct {
	plugin.NetRPCUnsupportedPlugin
	Impl Processor
}

func (p *ProcessorGRPCPlugin) GRPCServer(_ *plugin.GRPCBroker, s *grpc.Server) error {
	pb.RegisterProcessorServer(s, &grpcServer{impl: p.Impl})
	return nil
}

func (p *ProcessorGRPCPlugin) GRPCClient(_ context.Context, _ *plugin.GRPCBroker, c *grpc.ClientConn) (interface{}, error) {
	return &grpcClient{client: pb.NewProcessorClient(c)}, nil
}

type grpcServer struct {
	pb.UnimplementedProcessorServer
	impl Processor
}

func (s *grpcServer) Process(ctx context.Context, req *pb.ProcessRequest) (*pb.ProcessResponse, error) {
	out, err := s.impl.Process(ctx, ProcessRequest{Records: fromProto(req.GetRecords())})
	if err != nil {
		return nil, err
	}
	return &pb.ProcessResponse{Records: toProto(out.Records), Processed: out.Processed}, nil
}

type grpcClient struct {
	client pb.ProcessorClient
}

func (c *grpcClient) Process(ctx context.Context, req ProcessRequest) (ProcessResponse, error) {
	resp, err := c.client.Process(ctx, &pb.ProcessRequest{Records: toProto(req.Records)})
	if err != nil {
		return ProcessResponse{}, err
	}
	return ProcessResponse{Records: fromProto(resp.GetRecords()), Processed: resp.GetProcessed()}, nil
}

func toProto(rs []Record) []*pb.Record {
	out := make([]*pb.Record, len(rs))
	for i, r := range rs {
		out[i] = &pb.Record{Id: r.ID, Body: r.Body}
	}
	return out
}

func fromProto(rs []*pb.Record) []Record {
	out := make([]Record, len(rs))
	for i, r := range rs {
		out[i] = Record{ID: r.GetId(), Body: r.GetBody()}
	}
	return out
}
```

### The supervisor

`host.Manager` is the supervision surface. `New` builds a `plugin.Client` around
a `Cmd`, and the configuration is where the supervision policy lives:

- `AllowedProtocols: []plugin.Protocol{plugin.ProtocolGRPC}` pins the transport.
  Leave it unset and the client may fall back to net/rpc or refuse to negotiate,
  producing errors that are hard to read.
- `Managed: true` registers the client so `plugin.CleanupClients()` will reap it —
  a safety net for the case where you forget an explicit `Kill`.
- `StartTimeout` bounds how long `Client()` waits for the plugin to finish its
  handshake. A binary that hangs on start fails fast instead of blocking the host.
- `SkipHostEnv: true` keeps the host's environment out of the child, so the plugin
  runs with a clean, deliberate environment.
- `Logger` is an `hclog.Logger`; go-plugin routes the plugin's stdout/stderr and
  log output through it, so plugin diagnostics land in the host's structured log
  stream instead of a detached terminal.

The lifecycle methods map one-to-one onto process supervision. `Client()` launches
the process and performs the handshake (it is idempotent — later calls return the
same connection). `Dispense("processor")` returns the typed interface;
`asProcessor` isolates the type assertion so its failure path — `ErrWrongType` —
is unit-testable without a live connection. `Ping()` is the liveness probe.
`Exited()` reports whether the child has died. `NegotiatedVersion()` is the ABI
version the handshake settled on. `Close()` calls `Kill()` to terminate the child.

Create `host/host.go`:

```go
package host

import (
	"errors"
	"fmt"
	"os/exec"
	"time"

	hclog "github.com/hashicorp/go-hclog"
	plugin "github.com/hashicorp/go-plugin"

	"example.com/pluginhost/shared"
)

// ErrWrongType is returned when a dispensed value is not a shared.Processor,
// which means the plugin advertised the name but implements a different contract.
var ErrWrongType = errors.New("dispensed value is not a Processor")

// Manager supervises a single plugin process.
type Manager struct {
	client *plugin.Client
}

// New builds a supervised manager that launches the plugin binary at path. The
// logger is handed to plugin.NewClient so plugin diagnostics land in the host's
// structured log stream; the Manager keeps no logger state of its own.
func New(path string, log hclog.Logger) *Manager {
	client := plugin.NewClient(&plugin.ClientConfig{
		HandshakeConfig:  shared.Handshake,
		Plugins:          shared.PluginMap,
		Cmd:              exec.Command(path),
		AllowedProtocols: []plugin.Protocol{plugin.ProtocolGRPC},
		Managed:          true,
		StartTimeout:     10 * time.Second,
		SkipHostEnv:      true,
		Logger:           log,
	})
	return &Manager{client: client}
}

// newWithClient wraps an already-built client (used by tests over reattach).
func newWithClient(c *plugin.Client) *Manager {
	return &Manager{client: c}
}

// asProcessor isolates the type assertion so its failure path is testable.
func asProcessor(raw interface{}) (shared.Processor, error) {
	proc, ok := raw.(shared.Processor)
	if !ok {
		return nil, fmt.Errorf("dispensed %T: %w", raw, ErrWrongType)
	}
	return proc, nil
}

// Dispense connects (launching the process on first call) and returns the typed
// Processor.
func (m *Manager) Dispense() (shared.Processor, error) {
	rpc, err := m.client.Client()
	if err != nil {
		return nil, fmt.Errorf("connect plugin: %w", err)
	}
	raw, err := rpc.Dispense("processor")
	if err != nil {
		return nil, fmt.Errorf("dispense processor: %w", err)
	}
	return asProcessor(raw)
}

// Ping is the liveness probe. It returns nil while the plugin is healthy and a
// non-nil error once the connection is gone.
func (m *Manager) Ping() error {
	rpc, err := m.client.Client()
	if err != nil {
		return err
	}
	return rpc.Ping()
}

// Exited reports whether the plugin process has terminated.
func (m *Manager) Exited() bool { return m.client.Exited() }

// NegotiatedVersion is the ABI version the handshake settled on.
func (m *Manager) NegotiatedVersion() int { return m.client.NegotiatedVersion() }

// Close terminates the plugin process. Always call it; pair with
// plugin.CleanupClients in main to reap any client you forgot to close.
func (m *Manager) Close() { m.client.Kill() }
```

### The plugin binary

The plugin is the same PII redactor, serving over gRPC.

Create `plugin/main.go`:

```go
package main

import (
	"context"
	"regexp"

	plugin "github.com/hashicorp/go-plugin"

	"example.com/pluginhost/shared"
)

var cardRE = regexp.MustCompile(`\b\d{13,19}\b`)

type redactor struct{}

func (redactor) Process(_ context.Context, req shared.ProcessRequest) (shared.ProcessResponse, error) {
	if err := req.Validate(); err != nil {
		return shared.ProcessResponse{}, err
	}
	out := make([]shared.Record, len(req.Records))
	for i, r := range req.Records {
		out[i] = shared.Record{ID: r.ID, Body: cardRE.ReplaceAllString(r.Body, "[REDACTED]")}
	}
	return shared.ProcessResponse{Records: out, Processed: int64(len(out))}, nil
}

func main() {
	plugin.Serve(&plugin.ServeConfig{
		HandshakeConfig: shared.Handshake,
		Plugins:         plugin.PluginSet{"processor": &shared.ProcessorGRPCPlugin{Impl: redactor{}}},
		GRPCServer:      plugin.DefaultGRPCServer,
	})
}
```

### The runnable demo

Unlike Exercise 1, this demo launches the *real* plugin process, because
supervision is the point — you cannot supervise a goroutine. Build the plugin
first, then run the demo. The host logger is set to `Warn` so routine plugin
chatter stays out of stdout; the demo's own output is deterministic. Note the two
deferred cleanups: `m.Close()` kills this plugin, and `plugin.CleanupClients()`
reaps any managed client the process forgot to kill — the belt-and-suspenders that
keeps a long-running host from leaking children.

Create `cmd/demo/main.go`:

```go
package main

import (
	"context"
	"fmt"
	"os"

	hclog "github.com/hashicorp/go-hclog"
	plugin "github.com/hashicorp/go-plugin"

	"example.com/pluginhost/host"
	"example.com/pluginhost/shared"
)

func main() {
	log := hclog.New(&hclog.LoggerOptions{Name: "host", Level: hclog.Warn, Output: os.Stderr})

	m := host.New("./plugin/processor", log)
	defer plugin.CleanupClients()
	defer m.Close()

	proc, err := m.Dispense()
	if err != nil {
		fmt.Fprintln(os.Stderr, "dispense:", err)
		os.Exit(1)
	}
	fmt.Println("plugin dispensed: processor")

	if err := m.Ping(); err != nil {
		fmt.Fprintln(os.Stderr, "ping:", err)
		os.Exit(1)
	}
	fmt.Println("ping: ok")

	resp, err := proc.Process(context.Background(), shared.ProcessRequest{
		Records: []shared.Record{
			{ID: "r1", Body: "charge 4111111111111111 approved"},
			{ID: "r2", Body: "hello world"},
		},
	})
	if err != nil {
		fmt.Fprintln(os.Stderr, "process:", err)
		os.Exit(1)
	}
	fmt.Printf("processed %d records\n", resp.Processed)
	for _, r := range resp.Records {
		fmt.Printf("%s: %s\n", r.ID, r.Body)
	}
	fmt.Printf("alive: %v\n", !m.Exited())
}
```

Run it:

```bash
go build -o plugin/processor ./plugin
go run ./cmd/demo
```

Expected output:

```
plugin dispensed: processor
ping: ok
processed 2 records
r1: charge [REDACTED] approved
r2: hello world
alive: true
```

### Tests

The default tests use the reattach harness, so they are deterministic and need no
compiled binary. `TestDispenseAndProcess` serves a stub in-process, dispenses over
`Reattach`, and asserts a batch round-trips. `TestPingLifecycle` asserts `Ping` is
nil while the plugin is alive and non-nil after `Close` — the crash-detection
contract, exercised through an explicit teardown. `TestAsProcessor` unit-tests the
type-assertion seam directly, asserting `ErrWrongType` travels through `%w`.
`ExampleManager_dispenseWrongType` pins the exact wrapped-error text as a
verifiable `// Output` block.

Create `host/host_test.go`:

```go
package host

import (
	"context"
	"errors"
	"fmt"
	"testing"

	plugin "github.com/hashicorp/go-plugin"

	"example.com/pluginhost/shared"
)

// stubProc implements shared.Processor for the in-process server.
type stubProc struct{}

func (stubProc) Process(_ context.Context, req shared.ProcessRequest) (shared.ProcessResponse, error) {
	if err := req.Validate(); err != nil {
		return shared.ProcessResponse{}, err
	}
	return shared.ProcessResponse{Records: req.Records, Processed: int64(len(req.Records))}, nil
}

// serveReattach serves a stubProc in-process and returns a connected Manager.
func serveReattach(t *testing.T) *Manager {
	t.Helper()
	ch := make(chan *plugin.ReattachConfig, 1)
	closeCh := make(chan struct{})
	go plugin.Serve(&plugin.ServeConfig{
		HandshakeConfig: shared.Handshake,
		Plugins:         plugin.PluginSet{"processor": &shared.ProcessorGRPCPlugin{Impl: stubProc{}}},
		GRPCServer:      plugin.DefaultGRPCServer,
		Test:            &plugin.ServeTestConfig{ReattachConfigCh: ch, CloseCh: closeCh},
	})
	client := plugin.NewClient(&plugin.ClientConfig{
		HandshakeConfig:  shared.Handshake,
		Plugins:          shared.PluginMap,
		Reattach:         <-ch,
		AllowedProtocols: []plugin.Protocol{plugin.ProtocolGRPC},
	})
	m := newWithClient(client)
	t.Cleanup(m.Close)
	return m
}

func TestDispenseAndProcess(t *testing.T) {
	m := serveReattach(t)
	proc, err := m.Dispense()
	if err != nil {
		t.Fatalf("Dispense: %v", err)
	}
	resp, err := proc.Process(t.Context(), shared.ProcessRequest{Records: []shared.Record{{ID: "a"}, {ID: "b"}}})
	if err != nil {
		t.Fatalf("Process: %v", err)
	}
	if resp.Processed != 2 {
		t.Fatalf("Processed = %d, want 2", resp.Processed)
	}
}

func TestPingLifecycle(t *testing.T) {
	m := serveReattach(t)
	if _, err := m.Dispense(); err != nil {
		t.Fatalf("Dispense: %v", err)
	}
	if err := m.Ping(); err != nil {
		t.Fatalf("Ping while alive = %v, want nil", err)
	}
	m.Close()
	if err := m.Ping(); err == nil {
		t.Fatal("Ping after Close = nil, want a connection error")
	}
}

func TestAsProcessor(t *testing.T) {
	t.Parallel()
	if _, err := asProcessor(42); !errors.Is(err, ErrWrongType) {
		t.Fatalf("asProcessor(42) err = %v, want errors.Is ErrWrongType", err)
	}
	if _, err := asProcessor(stubProc{}); err != nil {
		t.Fatalf("asProcessor(Processor) = %v, want nil", err)
	}
}

// ExampleManager_dispenseWrongType shows the type-assertion seam: a dispensed
// value that is not a shared.Processor yields ErrWrongType, wrapped with the
// concrete type via %w, so errors.Is matches the sentinel.
func ExampleManager_dispenseWrongType() {
	_, err := asProcessor(42)
	fmt.Println(err)
	fmt.Println(errors.Is(err, ErrWrongType))
	// Output:
	// dispensed int: dispensed value is not a Processor
	// true
}
```

The integration test proves the two things reattach cannot: launching a real
binary end to end, and that a too-short `StartTimeout` against a process that never
handshakes surfaces a timeout error. It is tagged `manual` so it never runs in a
plain `go test`.

Create `host/host_integration_test.go`:

```go
//go:build manual

package host

import (
	"os/exec"
	"testing"
	"time"

	hclog "github.com/hashicorp/go-hclog"
	plugin "github.com/hashicorp/go-plugin"

	"example.com/pluginhost/shared"
)

func TestRealBinaryRoundTrip(t *testing.T) {
	m := New("../plugin/processor", hclog.NewNullLogger())
	defer m.Close()

	proc, err := m.Dispense()
	if err != nil {
		t.Fatalf("Dispense: %v", err)
	}
	resp, err := proc.Process(t.Context(), shared.ProcessRequest{
		Records: []shared.Record{{ID: "r1", Body: "pan 4111111111111111 ok"}},
	})
	if err != nil {
		t.Fatalf("Process: %v", err)
	}
	if resp.Records[0].Body != "pan [REDACTED] ok" {
		t.Fatalf("Body = %q, want redacted", resp.Records[0].Body)
	}
	if m.NegotiatedVersion() != 1 {
		t.Fatalf("NegotiatedVersion = %d, want 1", m.NegotiatedVersion())
	}
}

func TestStartTimeout(t *testing.T) {
	// A binary that never speaks the handshake: sleep. A 100ms StartTimeout must
	// fail Client() rather than block the host.
	client := plugin.NewClient(&plugin.ClientConfig{
		HandshakeConfig:  shared.Handshake,
		Plugins:          shared.PluginMap,
		Cmd:              exec.Command("sleep", "5"),
		AllowedProtocols: []plugin.Protocol{plugin.ProtocolGRPC},
		StartTimeout:     100 * time.Millisecond,
		Logger:           hclog.NewNullLogger(),
	})
	defer client.Kill()
	if _, err := client.Client(); err == nil {
		t.Fatal("Client() = nil error, want a start timeout")
	}
}
```

## Review

The supervisor is correct when the plugin behaves like any other supervised
service: `Client()` launches and hands back a connection, `Ping` reports health,
`Kill` tears down, and `CleanupClients` reaps whatever you forgot. Confirm the
happy path and crash detection with `TestDispenseAndProcess` and
`TestPingLifecycle` over reattach; confirm the real launch and startup bound with
the `manual`-tagged integration tests once the binary is built. If `Ping` never
turns non-nil after `Close`, the connection is being re-established somewhere it
should not be.

The mistakes to avoid are operational. Do not omit `AllowedProtocols` — an unset
value invites a net/rpc fallback and confusing negotiation errors. Do not forget
`defer m.Close()` and `defer plugin.CleanupClients()`; without them a host that
exits or is signaled orphans its children. Do not treat a plugin call as
infallible: it is an RPC, so a crash arrives as an error on the next call, not a
panic you can recover, which is exactly why `Ping` and a restart policy exist.
And keep plugin logs flowing through `hclog` — a plugin whose output goes nowhere
gives you no signal when a vendor binary misbehaves in production.
