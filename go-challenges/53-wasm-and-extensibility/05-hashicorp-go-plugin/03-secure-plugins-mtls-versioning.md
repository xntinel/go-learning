# Exercise 3: Securing Plugins: Checksum Pinning, AutoMTLS, and Version Negotiation

`NewClient` exec's a binary you did not compile. This exercise builds the trust
and compatibility layer around that fact: pin the executable with a SHA-256
`SecureConfig` so a tampered or swapped binary is refused before launch, enable
`AutoMTLS` so only the launching host can talk to the plugin socket, and use
`VersionedPlugins` to negotiate a compatible protocol version and fail with a
readable error on mismatch. It closes with the `GRPCBroker` pattern for injecting a
host capability back into the plugin.

This module is fully self-contained: it bundles the contract and adds the security
layer. It is bar-mode â€” it depends on `hashicorp/go-plugin`, `hclog`, `grpc`, and a
generated proto package, so it does not build offline. The checksum and
version-negotiation logic is pure and deterministically testable; AutoMTLS and the
broker callback are shown in shape and exercised behind a build tag.

## What you'll build

```text
pluginsecurity/                  independent module: example.com/pluginsecurity
  go.mod                         go 1.26; go-plugin, hclog, grpc, protobuf
  proto/
    processor.proto              service Processor { rpc Process }  (generated by protoc)
  shared/
    processor.go                 bundled contract + VersionedPlugins (v1, v2)
  secure/
    secure.go                    Checksum, SecureConfig, Negotiate/ErrVersionMismatch, SecureClientConfig, VersionSet
    secure_test.go               Check match/mismatch on temp files, Negotiate table (errors.Is), Example
  cmd/
    demo/
      main.go                    compute a checksum, accept a matching binary, reject a tampered one
```

Files: `shared/processor.go`, `secure/secure.go`, `secure/secure_test.go`, `cmd/demo/main.go`.
Implement: `Checksum` (SHA-256 of a file), `SecureConfig` (pin), `Negotiate` (highest common protocol version) with a sentinel `ErrVersionMismatch`, and `SecureClientConfig` wiring `SecureConfig` + `AutoMTLS` + `VersionedPlugins`.
Test: `SecureConfig.Check` returns true for a matching file and false for a mutated one; `Negotiate` picks the highest common version and returns `ErrVersionMismatch` (via `errors.Is`) when the sets are disjoint.
Verify: `go test -race ./...` (checksum + negotiation are pure and deterministic); the AutoMTLS launch path runs against a built binary.

### The contract, plus versioned plugin sets

The bundled contract is the same one from the earlier exercises, with one
addition: `VersionedPlugins`, a `map[int]plugin.PluginSet`. A host that sets
`VersionedPlugins` on both `ClientConfig` and `ServeConfig` can speak several ABI
versions and negotiate the highest the other side also supports. Here versions 1
and 2 both map to `ProcessorGRPCPlugin` â€” in a real rollout version 2 would carry a
changed proto or interface, and old plugins that only register version 1 would
still connect at version 1 instead of failing.

Create `shared/processor.go`:

```go
package shared

import (
	"context"
	"errors"
	"fmt"

	plugin "github.com/hashicorp/go-plugin"
	"google.golang.org/grpc"

	pb "example.com/pluginsecurity/proto"
)

// ErrEmptyBatch is returned by Validate when a request carries no records.
var ErrEmptyBatch = errors.New("processor: empty batch")

// Record is a domain record handed to the plugin.
type Record struct {
	ID   string
	Body string
}

// ProcessRequest is a coarse-grained batch.
type ProcessRequest struct {
	Records []Record
}

// Validate rejects an empty batch, wrapping ErrEmptyBatch with %w.
func (r ProcessRequest) Validate() error {
	if len(r.Records) == 0 {
		return fmt.Errorf("validate request: %w", ErrEmptyBatch)
	}
	return nil
}

// ProcessResponse carries transformed records and a processed count.
type ProcessResponse struct {
	Records   []Record
	Processed int64
}

// Processor is the domain interface host and plugin share.
type Processor interface {
	Process(ctx context.Context, req ProcessRequest) (ProcessResponse, error)
}

// Handshake gates the connection: magic cookie plus a base ABI version.
var Handshake = plugin.HandshakeConfig{
	ProtocolVersion:  1,
	MagicCookieKey:   "PROCESSOR_PLUGIN",
	MagicCookieValue: "8f2a1c7e-processor-v1",
}

// PluginMap is the single-version plugin set.
var PluginMap = plugin.PluginSet{
	"processor": &ProcessorGRPCPlugin{},
}

// VersionedPlugins lets host and plugin negotiate a compatible ABI version.
var VersionedPlugins = map[int]plugin.PluginSet{
	1: {"processor": &ProcessorGRPCPlugin{}},
	2: {"processor": &ProcessorGRPCPlugin{}},
}

// ProcessorGRPCPlugin bridges Processor to gRPC.
type ProcessorGRPCPlugin struct {
	plugin.NetRPCUnsupportedPlugin
	Impl Processor
}

func (p *ProcessorGRPCPlugin) GRPCServer(_ *plugin.GRPCBroker, s *grpc.Server) error {
	pb.RegisterProcessorServer(s, &grpcServer{impl: p.Impl})
	return nil
}

func (p *ProcessorGRPCPlugin) GRPCClient(_ context.Context, _ *plugin.GRPCBroker, c *grpc.ClientConn) (interface{}, error) {
	return &grpcClient{client: pb.NewProcessorClient(c)}, nil
}

type grpcServer struct {
	pb.UnimplementedProcessorServer
	impl Processor
}

func (s *grpcServer) Process(ctx context.Context, req *pb.ProcessRequest) (*pb.ProcessResponse, error) {
	out, err := s.impl.Process(ctx, ProcessRequest{Records: fromProto(req.GetRecords())})
	if err != nil {
		return nil, err
	}
	return &pb.ProcessResponse{Records: toProto(out.Records), Processed: out.Processed}, nil
}

type grpcClient struct {
	client pb.ProcessorClient
}

func (c *grpcClient) Process(ctx context.Context, req ProcessRequest) (ProcessResponse, error) {
	resp, err := c.client.Process(ctx, &pb.ProcessRequest{Records: toProto(req.Records)})
	if err != nil {
		return ProcessResponse{}, err
	}
	return ProcessResponse{Records: fromProto(resp.GetRecords()), Processed: resp.GetProcessed()}, nil
}

func toProto(rs []Record) []*pb.Record {
	out := make([]*pb.Record, len(rs))
	for i, r := range rs {
		out[i] = &pb.Record{Id: r.ID, Body: r.Body}
	}
	return out
}

func fromProto(rs []*pb.Record) []Record {
	out := make([]Record, len(rs))
	for i, r := range rs {
		out[i] = Record{ID: r.GetId(), Body: r.GetBody()}
	}
	return out
}
```

### The security layer

`secure.go` holds the three defenses. `Checksum` reads the file and returns its
SHA-256 â€” the digest you pin. `SecureConfig` wraps that digest in a
`*plugin.SecureConfig`; when the host launches, go-plugin recomputes the file's
hash and refuses to exec if it differs, so a swapped or tampered binary never runs.
One sharp edge worth internalizing: `SecureConfig.Check` does **not** reset its
`hash.Hash`, so a single `SecureConfig` is good for exactly one `Check`. Build a
fresh one per check (`SecureConfig(sum)` returns a fresh `sha256.New()` each time).

`Negotiate` is a pure preflight compatibility check: given the version sets the
host and the plugin advertise, it returns the highest they share, or
`ErrVersionMismatch` when they share none. This mirrors what go-plugin does at the
handshake (the settled value is later readable via `(*Client).NegotiatedVersion()`);
running it as a pure function lets you test the rule deterministically and lets the
host reject an incompatible plugin with a clear message before paying for a launch.

`SecureClientConfig` assembles the hardened launch config. `AutoMTLS: true` makes
go-plugin mint a one-time certificate pair for this launch and require mutual TLS on
the socket, so no other local process can connect to the plugin. It is mutually
exclusive with `Reattach` and with a manual `TLSConfig` â€” which is exactly why the
deterministic tests use the reattach harness and never touch AutoMTLS.

Create `secure/secure.go`:

```go
package secure

import (
	"crypto/sha256"
	"errors"
	"fmt"
	"io"
	"os"
	"os/exec"

	hclog "github.com/hashicorp/go-hclog"
	plugin "github.com/hashicorp/go-plugin"

	"example.com/pluginsecurity/shared"
)

// ErrVersionMismatch is returned when host and plugin share no protocol version.
var ErrVersionMismatch = errors.New("no compatible protocol version")

// Checksum returns the SHA-256 digest of the file at path.
func Checksum(path string) ([]byte, error) {
	f, err := os.Open(path)
	if err != nil {
		return nil, fmt.Errorf("open plugin: %w", err)
	}
	defer f.Close()

	h := sha256.New()
	if _, err := io.Copy(h, f); err != nil {
		return nil, fmt.Errorf("hash plugin: %w", err)
	}
	return h.Sum(nil), nil
}

// SecureConfig pins the plugin binary to sum. NewClient recomputes the file's
// hash at launch and refuses to exec a binary whose digest does not match.
// Each SecureConfig is single-use: Check does not reset the hash, so build a
// fresh one per check.
func SecureConfig(sum []byte) *plugin.SecureConfig {
	return &plugin.SecureConfig{Checksum: sum, Hash: sha256.New()}
}

// Negotiate returns the highest protocol version offered by both host and guest,
// or ErrVersionMismatch if they share none.
func Negotiate(host, guest map[int]struct{}) (int, error) {
	best := -1
	for v := range host {
		if _, ok := guest[v]; ok && v > best {
			best = v
		}
	}
	if best < 0 {
		return 0, fmt.Errorf("negotiate: %w", ErrVersionMismatch)
	}
	return best, nil
}

// VersionSet reduces a VersionedPlugins map to the set of versions it offers.
func VersionSet(vp map[int]plugin.PluginSet) map[int]struct{} {
	out := make(map[int]struct{}, len(vp))
	for v := range vp {
		out[v] = struct{}{}
	}
	return out
}

// SecureClientConfig builds a hardened launch config: checksum-pinned, AutoMTLS,
// and negotiating the highest common protocol version via VersionedPlugins.
// AutoMTLS is incompatible with Reattach and with a manual TLSConfig.
func SecureClientConfig(path string, sum []byte, log hclog.Logger) *plugin.ClientConfig {
	return &plugin.ClientConfig{
		HandshakeConfig:  shared.Handshake,
		VersionedPlugins: shared.VersionedPlugins,
		Cmd:              exec.Command(path),
		SecureConfig:     SecureConfig(sum),
		AutoMTLS:         true,
		AllowedProtocols: []plugin.Protocol{plugin.ProtocolGRPC},
		Managed:          true,
		Logger:           log,
	}
}
```

### Injecting a host capability with GRPCBroker

The default flow is host-calls-plugin, but a plugin usually needs something from
the host â€” an audit sink, a secret, a storage handle. Handing it ambient access
would defeat the isolation. `GRPCBroker` is the disciplined channel: the host
allocates a stream id with `NextId()`, serves a small gRPC service on it with
`AcceptAndServe`, and passes the id to the plugin inside a request; the plugin
`Dial`s that id and calls back. In the bridge (`GRPCServer`/`GRPCClient`) the
`*plugin.GRPCBroker` argument you have been ignoring is what makes this work. The
shape, host side:

```go
// Illustrative: the host serves an audit capability the plugin can call back into.
brokerID := broker.NextId()
go broker.AcceptAndServe(brokerID, func(opts []grpc.ServerOption) *grpc.Server {
	s := grpc.NewServer(opts...)
	auditpb.RegisterAuditServer(s, &auditServer{})
	return s
})
// send brokerID to the plugin in the request; the plugin dials it back:
conn, err := broker.Dial(brokerID)
```

Because the capability is a gRPC service you define and register, the plugin can
do exactly the operations you exposed and nothing more â€” a capability-scoped host
API rather than a blank check.

### The runnable demo

The demo makes the supply-chain check concrete and deterministic without launching
anything: it writes a known binary payload to a temp file, computes its checksum,
confirms a matching `SecureConfig` accepts it, then tampers with the file and
confirms a fresh `SecureConfig` rejects it. This is precisely the decision
`NewClient` makes at launch, run in isolation so you can watch both outcomes.

Create `cmd/demo/main.go`:

```go
package main

import (
	"encoding/hex"
	"fmt"
	"os"
	"path/filepath"

	"example.com/pluginsecurity/secure"
)

func main() {
	dir, err := os.MkdirTemp("", "plugindemo")
	if err != nil {
		fmt.Fprintln(os.Stderr, err)
		os.Exit(1)
	}
	defer os.RemoveAll(dir)

	path := filepath.Join(dir, "processor")
	if err := os.WriteFile(path, []byte("processor-plugin-v1"), 0o755); err != nil {
		fmt.Fprintln(os.Stderr, err)
		os.Exit(1)
	}

	sum, err := secure.Checksum(path)
	if err != nil {
		fmt.Fprintln(os.Stderr, err)
		os.Exit(1)
	}
	fmt.Printf("sha256: %s\n", hex.EncodeToString(sum))

	ok, _ := secure.SecureConfig(sum).Check(path)
	fmt.Printf("matching binary accepted: %v\n", ok)

	if err := os.WriteFile(path, []byte("processor-plugin-EVIL"), 0o755); err != nil {
		fmt.Fprintln(os.Stderr, err)
		os.Exit(1)
	}
	ok, _ = secure.SecureConfig(sum).Check(path)
	fmt.Printf("tampered binary accepted: %v\n", ok)
}
```

Run it:

```bash
go run ./cmd/demo
```

Expected output:

```
sha256: 31aae23b6b8cbab067176cc596128e43b2465a13fe1dd17be1a0a8ba658d0689
matching binary accepted: true
tampered binary accepted: false
```

### Tests

Both defenses test deterministically with no process launch. `TestCheck` writes a
file, pins its checksum, and asserts a fresh `SecureConfig` accepts it and rejects
a mutated copy â€” the exact preflight go-plugin runs. `TestNegotiate` drives the
version rule as a table, asserting the highest common version is chosen and that
disjoint sets yield `ErrVersionMismatch` through `errors.Is`. The `Example`
computes and formats a checksum.

Create `secure/secure_test.go`:

```go
package secure

import (
	"encoding/hex"
	"errors"
	"fmt"
	"os"
	"path/filepath"
	"testing"

	plugin "github.com/hashicorp/go-plugin"

	"example.com/pluginsecurity/shared"
)

func TestCheck(t *testing.T) {
	t.Parallel()
	dir := t.TempDir()
	path := filepath.Join(dir, "processor")
	if err := os.WriteFile(path, []byte("processor-plugin-v1"), 0o755); err != nil {
		t.Fatal(err)
	}

	sum, err := Checksum(path)
	if err != nil {
		t.Fatalf("Checksum: %v", err)
	}

	// A fresh SecureConfig per Check: Check does not reset the hash.
	ok, err := SecureConfig(sum).Check(path)
	if err != nil {
		t.Fatalf("Check(matching): %v", err)
	}
	if !ok {
		t.Fatal("matching binary was rejected")
	}

	if err := os.WriteFile(path, []byte("tampered"), 0o755); err != nil {
		t.Fatal(err)
	}
	ok, err = SecureConfig(sum).Check(path)
	if err != nil {
		t.Fatalf("Check(tampered): %v", err)
	}
	if ok {
		t.Fatal("tampered binary was accepted")
	}
}

func TestNegotiate(t *testing.T) {
	t.Parallel()
	tests := []struct {
		name    string
		host    map[int]struct{}
		guest   map[int]struct{}
		want    int
		wantErr error
	}{
		{"both v1 v2", VersionSet(shared.VersionedPlugins), map[int]struct{}{1: {}, 2: {}}, 2, nil},
		{"guest only v1", VersionSet(shared.VersionedPlugins), map[int]struct{}{1: {}}, 1, nil},
		{"disjoint", map[int]struct{}{2: {}}, map[int]struct{}{1: {}}, 0, ErrVersionMismatch},
	}
	for _, tc := range tests {
		t.Run(tc.name, func(t *testing.T) {
			t.Parallel()
			got, err := Negotiate(tc.host, tc.guest)
			if tc.wantErr != nil {
				if !errors.Is(err, tc.wantErr) {
					t.Fatalf("Negotiate err = %v, want errors.Is %v", err, tc.wantErr)
				}
				return
			}
			if err != nil {
				t.Fatalf("Negotiate: %v", err)
			}
			if got != tc.want {
				t.Fatalf("Negotiate = %d, want %d", got, tc.want)
			}
		})
	}
}

func TestVersionSet(t *testing.T) {
	t.Parallel()
	got := VersionSet(map[int]plugin.PluginSet{1: nil, 2: nil})
	if len(got) != 2 {
		t.Fatalf("VersionSet size = %d, want 2", len(got))
	}
}

func ExampleChecksum() {
	dir, _ := os.MkdirTemp("", "ex")
	defer os.RemoveAll(dir)
	path := filepath.Join(dir, "bin")
	_ = os.WriteFile(path, []byte("processor-plugin-v1"), 0o644)

	sum, _ := Checksum(path)
	fmt.Println(hex.EncodeToString(sum))
	// Output: 31aae23b6b8cbab067176cc596128e43b2465a13fe1dd17be1a0a8ba658d0689
}
```

## Review

The security layer is correct when a tampered binary cannot launch and an
incompatible plugin cannot connect. Confirm the first with `TestCheck`: a matching
file is accepted and any mutation is rejected, which is the exact digest comparison
go-plugin performs before `exec`. Confirm the second with `TestNegotiate`: the
highest shared version wins and disjoint sets produce `ErrVersionMismatch`. If
`Check` ever accepts a mutated file, the most likely cause is reusing one
`SecureConfig` across two checks â€” its hash is not reset between calls, so build a
fresh one each time.

The mistakes to avoid are the production ones. Do not ship without both defenses:
an unpinned binary is a supply-chain hole and an unauthenticated local socket is a
lateral-movement hole, and `SecureConfig` plus `AutoMTLS` close them. Do not try to
combine `AutoMTLS` with `Reattach` or a manual `TLSConfig` â€” they are mutually
exclusive and fail at connection time, which is why the deterministic tests avoid
AutoMTLS and use reattach. And do not change the proto or interface without adding a
`VersionedPlugins` entry and bumping the version; negotiation is what turns an
incompatible plugin into a readable error at the handshake instead of a corrupt
unmarshal at runtime. Run `go test -race ./...` to confirm the checksum and
negotiation logic.
