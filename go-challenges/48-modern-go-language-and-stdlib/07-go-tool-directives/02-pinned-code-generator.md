# Exercise 2: Build a Deterministic Code Generator to Pin with go tool

A tool directive pins the *version* of a generator; it cannot make the
generator's *output* deterministic. This exercise builds the kind of generator
you would pin — a stringer-style tool that emits a `String()` method for a typed
enum — and makes it deterministic by construction, so the artifact it produces
never diffs regardless of how the pinning is done.

This module is fully self-contained: its own `go mod init`, demo, and tests,
importing no other exercise.

## What you'll build

```text
enumgen/                      independent module: example.com/enumgen
  go.mod                      go 1.25
  enumgen.go                  Generate([]byte,string) ([]byte,error); ErrParse, ErrNoConsts
  cmd/
    demo/
      main.go                 generates a String method for a sample enum to stdout
  enumgen_test.go             golden output, valid-Go re-parse, determinism, Example
```

- Files: `enumgen.go`, `cmd/demo/main.go`, `enumgen_test.go`.
- Implement: `Generate(src []byte, typeName string) ([]byte, error)` that parses Go source, collects the constants of `typeName` (honoring `iota` blocks), and emits a gofmt-formatted `String()` method with cases in sorted order.
- Test: a golden-file comparison, a re-parse of the emitted code to prove it is valid Go, a determinism check that two runs are byte-identical, and an `iota`-inheritance case.
- Verify: `go test -count=1 -race ./...`

Set up the module:

```bash

```

### The pinning is the point, the determinism is the responsibility

Pin this generator with `go get -tool example.com/enumgen` and drive it from a
`//go:generate go tool enumgen -type=Status` line, and every developer builds the
same binary from the same source. That guarantee is worthless if the generator
emits map-ordered output: the pinned tool would still produce a different file on
each machine, and `git status` would never be clean after `go generate ./...`.
So the generator sorts its constant names before emitting, making the output a
pure function of the input source and the type name.

### Parsing: the iota inheritance subtlety

`go/parser.ParseFile` returns an `*ast.File` whose `Decls` include the top-level
declarations; a `const` block is a `*ast.GenDecl` with `Tok == token.CONST` and
one `*ast.ValueSpec` per line. The subtle part is `iota` grouping. In

```go
const (
	Sunny Condition = iota
	Cloudy
	Rainy
)
```

only the first spec carries `Type` (`Condition`); `Cloudy` and `Rainy` have a
`nil` `Type` and inherit it. The Go spec rule is precise: within a parenthesized
const block, a spec with an *empty expression list* repeats the type and value of
the last spec that *had* an expression list. So `collectConsts` tracks a
`lastType`: whenever a spec has a non-empty `Values` list it resets `lastType`
(to the explicit type if there is one, else empty because the type is inferred
and does not carry forward), and whenever a spec has an empty `Values` list it
inherits the current `lastType`. A constant is collected exactly when `lastType`
equals the requested type. This is why an untyped `const Debug = 99` in a
separate block is never collected: its own expression list resets `lastType` to
empty. Blank identifiers (`_`) are skipped so gaps in an enum do not emit cases.

### Emitting: template then gofmt then re-parse

The method is rendered with `text/template`, then run through
`go/format.Source`, which is the canonical gofmt pass over generated bytes — it
guarantees the output is exactly what `gofmt` would produce, tabs and all. The
default case falls back to `strconv.Itoa(int(v))` so an out-of-range value
stringifies as `Status(7)` rather than the empty string, matching what the real
`stringer` does. Two sentinels distinguish failure modes: `ErrParse` when the
source will not parse, and `ErrNoConsts` when the type has no constants (a likely
typo in `-type`), both wrapped with `%w`.

Create `enumgen.go`:

```go
package enumgen

import (
	"bytes"
	"errors"
	"fmt"
	"go/ast"
	"go/format"
	"go/parser"
	"go/token"
	"sort"
	"text/template"
)

var (
	ErrParse    = errors.New("enumgen: parse source")
	ErrNoConsts = errors.New("enumgen: no constants of type")
)

var tmpl = template.Must(template.New("stringer").Parse(`// Code generated by enumgen; DO NOT EDIT.

package {{.Package}}

import "strconv"

func (v {{.Type}}) String() string {
	switch v {
{{- range .Names}}
	case {{.}}:
		return "{{.}}"
{{- end}}
	default:
		return "{{.Type}}(" + strconv.Itoa(int(v)) + ")"
	}
}
`))

type tmplData struct {
	Package string
	Type    string
	Names   []string
}

// Generate reads Go source and emits a *_string.go body with a String method
// for every constant declared with the named type. Output is deterministic:
// constant names are sorted, so regenerating never produces a diff.
func Generate(src []byte, typeName string) ([]byte, error) {
	fset := token.NewFileSet()
	f, err := parser.ParseFile(fset, "input.go", src, 0)
	if err != nil {
		return nil, fmt.Errorf("%w: %v", ErrParse, err)
	}

	names := collectConsts(f, typeName)
	if len(names) == 0 {
		return nil, fmt.Errorf("%w %q", ErrNoConsts, typeName)
	}
	sort.Strings(names)

	var buf bytes.Buffer
	if err := tmpl.Execute(&buf, tmplData{Package: f.Name.Name, Type: typeName, Names: names}); err != nil {
		return nil, err
	}
	return format.Source(buf.Bytes())
}

// collectConsts returns the identifiers of every constant whose type resolves
// to typeName, honoring iota blocks where only the first spec carries the type.
func collectConsts(f *ast.File, typeName string) []string {
	var names []string
	for _, decl := range f.Decls {
		gd, ok := decl.(*ast.GenDecl)
		if !ok || gd.Tok != token.CONST {
			continue
		}
		var lastType string
		for _, spec := range gd.Specs {
			vs := spec.(*ast.ValueSpec)
			if len(vs.Values) > 0 {
				// A non-empty expression list resets the inherited type.
				lastType = ""
				if id, ok := vs.Type.(*ast.Ident); ok {
					lastType = id.Name
				}
			}
			// An empty expression list inherits lastType (and value) from above.
			if lastType == typeName {
				for _, name := range vs.Names {
					if name.Name != "_" {
						names = append(names, name.Name)
					}
				}
			}
		}
	}
	return names
}
```

### The runnable demo

Create `cmd/demo/main.go`:

```go
package main

import (
	"fmt"
	"os"

	"example.com/enumgen"
)

const sample = `package payment

type Status int

const (
	Pending Status = iota
	Authorized
	Captured
	Refunded
)
`

func main() {
	out, err := enumgen.Generate([]byte(sample), "Status")
	if err != nil {
		fmt.Fprintln(os.Stderr, "generate:", err)
		os.Exit(1)
	}
	os.Stdout.Write(out)
}
```

Run it:

```bash
go run ./cmd/demo
```

Expected output:

```
// Code generated by enumgen; DO NOT EDIT.

package payment

import "strconv"

func (v Status) String() string {
	switch v {
	case Authorized:
		return "Authorized"
	case Captured:
		return "Captured"
	case Pending:
		return "Pending"
	case Refunded:
		return "Refunded"
	default:
		return "Status(" + strconv.Itoa(int(v)) + ")"
	}
}
```

### Tests

`TestGenerateGolden` compares against an exact expected file. `TestGeneratedIsValidGo`
feeds the emitted bytes back through `parser.ParseFile`, proving the output is
syntactically valid Go rather than merely string-equal to a golden — the
strongest check a code generator can have. `TestDeterministic` runs generation
twice and asserts byte-identical output. `TestInheritedTypeAcrossIota` pins the
`iota` rule: `Info`/`Warn`/`Error` inherit the `Level` type and are collected,
while an untyped `Debug` in a later block is not. `TestErrors` covers both
sentinels.

Create `enumgen_test.go`:

```go
package enumgen

import (
	"errors"
	"fmt"
	"go/parser"
	"go/token"
	"testing"
)

const sampleSrc = `package weather

type Condition int

const (
	Sunny Condition = iota
	Cloudy
	Rainy
	Stormy
)
`

const goldenOut = `// Code generated by enumgen; DO NOT EDIT.

package weather

import "strconv"

func (v Condition) String() string {
	switch v {
	case Cloudy:
		return "Cloudy"
	case Rainy:
		return "Rainy"
	case Stormy:
		return "Stormy"
	case Sunny:
		return "Sunny"
	default:
		return "Condition(" + strconv.Itoa(int(v)) + ")"
	}
}
`

func TestGenerateGolden(t *testing.T) {
	t.Parallel()
	got, err := Generate([]byte(sampleSrc), "Condition")
	if err != nil {
		t.Fatalf("Generate: %v", err)
	}
	if string(got) != goldenOut {
		t.Errorf("Generate output:\n%s\nwant:\n%s", got, goldenOut)
	}
}

func TestGeneratedIsValidGo(t *testing.T) {
	t.Parallel()
	got, err := Generate([]byte(sampleSrc), "Condition")
	if err != nil {
		t.Fatalf("Generate: %v", err)
	}
	fset := token.NewFileSet()
	if _, err := parser.ParseFile(fset, "out.go", got, 0); err != nil {
		t.Fatalf("emitted code does not parse: %v", err)
	}
}

func TestDeterministic(t *testing.T) {
	t.Parallel()
	a, err := Generate([]byte(sampleSrc), "Condition")
	if err != nil {
		t.Fatalf("Generate: %v", err)
	}
	b, err := Generate([]byte(sampleSrc), "Condition")
	if err != nil {
		t.Fatalf("Generate: %v", err)
	}
	if string(a) != string(b) {
		t.Error("two runs produced different output")
	}
}

func TestErrors(t *testing.T) {
	t.Parallel()
	if _, err := Generate([]byte("package p\n\ntype T int\n"), "T"); !errors.Is(err, ErrNoConsts) {
		t.Errorf("missing consts: err = %v; want ErrNoConsts", err)
	}
	if _, err := Generate([]byte("this is not go"), "T"); !errors.Is(err, ErrParse) {
		t.Errorf("bad source: err = %v; want ErrParse", err)
	}
}

func TestInheritedTypeAcrossIota(t *testing.T) {
	t.Parallel()
	// Only Level carries an explicit type; Warn/Error inherit it, while the
	// untyped Debug in a later block must NOT be collected.
	const src = `package log

type Level int

const (
	Info Level = iota
	Warn
	Error
)

const Debug = 99
`
	got, err := Generate([]byte(src), "Level")
	if err != nil {
		t.Fatalf("Generate: %v", err)
	}
	for _, want := range []string{"case Info:", "case Warn:", "case Error:"} {
		if !contains(string(got), want) {
			t.Errorf("output missing %q:\n%s", want, got)
		}
	}
	if contains(string(got), "Debug") {
		t.Errorf("untyped Debug wrongly collected:\n%s", got)
	}
}

func contains(s, sub string) bool {
	for i := 0; i+len(sub) <= len(s); i++ {
		if s[i:i+len(sub)] == sub {
			return true
		}
	}
	return false
}

func ExampleGenerate() {
	src := `package p

type Suit int

const (
	Spades Suit = iota
	Hearts
)
`
	out, _ := Generate([]byte(src), "Suit")
	fmt.Print(string(out))
	// Output:
	// // Code generated by enumgen; DO NOT EDIT.
	//
	// package p
	//
	// import "strconv"
	//
	// func (v Suit) String() string {
	// 	switch v {
	// 	case Hearts:
	// 		return "Hearts"
	// 	case Spades:
	// 		return "Spades"
	// 	default:
	// 		return "Suit(" + strconv.Itoa(int(v)) + ")"
	// 	}
	// }
}
```

You would wire the generated method into a type with a directive like:

```go
//go:generate go tool enumgen -type=Status
```

so that `go generate ./...` runs the *pinned* generator, not whatever `stringer`
happens to be on `PATH`.

## Review

The generator is correct when its output is a pure function of `(src, typeName)`:
the golden test fixes the exact bytes, the re-parse test proves those bytes
compile, and the determinism test proves repeated runs never diverge. If the
determinism test ever flakes, something non-deterministic slipped in — an
unsorted map, an environment read — and no amount of tool-directive pinning would
save the artifact from diffing.

The traps: forgetting the `iota` inheritance rule and collecting only the first
constant of a block (or, worse, collecting an untyped constant from a later
block); emitting without `go/format.Source`, so the output is not gofmt-stable;
and omitting the `default` case, so out-of-range values stringify to `""`. Run
`go test -race` to confirm; the generator holds no shared state, so it is
trivially race-free, and the test proves it.
